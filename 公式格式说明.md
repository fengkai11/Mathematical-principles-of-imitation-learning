# 公式格式说明

这份说明用于约束网页和文稿中的公式修改，避免 GPT 把数学表达式改坏。

## 基本规则

1. 行内公式使用 `\( ... \)`。
2. 独立公式使用 `$$ ... $$`，并且前后各空一行。
3. 多行推导使用 `aligned` 或 `align` 环境。
4. 公式中的变量、上下标、希腊字母、条件概率符号、范数符号不要改写成中文。
5. 修改正文时，只改公式外的中文说明，不要改公式边界和数学符号。

## 行内公式

行内公式用于嵌在句子中，例如：

```markdown
模仿学习可以写成 \( \pi_\theta(a \mid o) \)。
```

注意：

- 不要把 `\( ... \)` 改成普通括号。
- 不要把公式拆成普通文本。
- 不要在公式内部随意加中文字符。

## 独立公式

独立公式用于单独成段展示，例如：

```markdown
$$
\mathcal{L} = \mathbb{E}[\|a - \hat a\|^2]
$$
```

注意：

- 不要把 `$$ ... $$` 改成行内文本。
- 不要把独立公式塞进句子里。
- 不要省略前后空行。

## 多行推导

多行推导用于展示一组对齐的等式，例如：

```markdown
$$
\begin{aligned}
x &= a + b \\
y &= c + d
\end{aligned}
$$
```

注意：

- 需要换行对齐时，用 `aligned` 或 `align`。
- 不要用普通换行替代数学环境。
- 不要把对齐符号 `&` 删除。

## 常见错误

- 把 `\( ... \)` 改成普通括号。
- 把 `$$ ... $$` 拆成普通文字段落。
- 把 `a_t`、`o_t`、`\theta`、`\tau` 改成中文描述。
- 把条件概率写法 `\pi(a \mid o)` 改坏。
- 把独立公式硬塞进列表句子里，导致渲染混乱。

## 给 GPT 的约束句

```text
保留所有公式分隔符和数学符号，不要把 \( ... \) 改成普通括号，不要把 $$...$$ 改成行内文本，不要改变量名、上下标、希腊字母和条件概率写法；如需改写，只改公式外的中文说明。
```

## 最稳的写法

正文叙述保持中文，公式只在数学分隔符里写 TeX。这样 MathJax 才能稳定渲染，GPT 也不容易改坏公式格式。
