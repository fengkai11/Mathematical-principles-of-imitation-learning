# 成书级章节改造方法

> 本文档用于指导《模仿学习的数学原理》后续章节的系统性改造。它总结了第 1 章从“普通解释稿”改造成“成书级概念入口章”的方法，后续第 2 章 Behavior Cloning 以及其他章节都可以按此规范检查和重写。

---

## 1. 改造目标

成书级章节改造不是简单润色，也不是扩写字数，而是让每一章都具备以下能力：

1. **概念递进清楚**：先定义基础对象，再引入依赖它们的复杂对象。
2. **数学符号清楚**：第一次出现的变量、公式、条件概率、损失函数都要解释。
3. **工程含义清楚**：每个公式都要能落回机械臂抓取与放置主线任务中理解。
4. **章节结构稳定**：章标题、小节标题、公式索引、定义索引、思考题、推荐阅读形成统一格式。
5. **mdBook 显示稳定**：公式、图片、表格、列表都要适合网页阅读。
6. **读者路径友好**：面向只有基础数学知识的算法工程师，不假设读者已经熟悉概率统计、强化学习或机器人学习背景。

一句话概括：

```text
先让读者知道问题从哪里来，再定义概念，再写公式，再拆公式，再落到工程任务，最后指出局限和下一章问题。
```

---

## 2. 总体改造原则

### 2.1 从“问题罗列”改为“概念依赖链”

章节小节顺序必须遵守概念依赖关系。

例如第 1 章最终采用的顺序是：

```text
任务
→ 观测与状态
→ 动作与动作空间
→ 观测—动作样本
→ 轨迹
→ 示范数据集
→ 策略
→ 专家策略
→ 闭环执行
```

这样可以避免出现下面的问题：

```text
轨迹公式里已经出现 o_t 和 a_t，
但观测和动作还没有正式定义。
```

后续章节同理。比如第 2 章 Behavior Cloning 应该遵守：

```text
第一章留下的问题
→ 行为克隆定义
→ 训练样本
→ 策略模型形式
→ 损失函数
→ 经验风险最小化
→ 机械臂训练流程
→ 闭环执行风险
→ 本章总结
```

---

### 2.2 第一次出现的重要概念必须正式定义

不能只用描述性文字带过。凡是第一次出现、后续会反复使用的概念，都应该用“定义 X.X”显式给出。

定义格式建议：

```markdown
> **定义 2.1：行为克隆**
>
> 行为克隆（Behavior Cloning, BC）是指把专家示范中的观测—动作样本看作监督学习数据，训练一个策略模型，使其在给定观测时输出接近专家动作的动作。
```

定义之后必须接解释：

```text
这个定义里有三个关键词：观测—动作样本、监督学习、策略模型。
```

定义不宜过长。定义负责“准确”，后面的段落负责“解释”。

---

### 2.3 公式必须按“动机—公式—读法—符号—直觉—工程含义”拆解

核心公式不能直接抛给读者。

推荐结构：

```markdown
**公式 (2.1)：行为克隆的经验风险目标**

$$
\mathcal{L}_{BC}(\theta)
= \frac{1}{N}\sum_{i=1}^{N}\ell(f_\theta(o_i), a_i)
$$

这个公式读作：……

其中：

- $\theta$：……
- $N$：……
- $o_i$：……
- $a_i$：……
- $f_\theta(o_i)$：……
- $\ell(\cdot, \cdot)$：……

直觉上，它是在做一件事：……

放到机械臂任务中，它表示：……

常见误解是：……
```

所有重要公式都要尽量遵守这个结构，尤其是：

- 策略公式；
- 损失函数；
- 概率分布；
- 最大似然；
- 负对数似然；
- 期望；
- KL 散度；
- occupancy measure；
- diffusion / flow matching 相关公式。

## 3. 公式格式规范

仓库中已有 `公式格式说明.md`，后续章节必须优先遵守该文档。

结合当前 mdBook 实际显示情况，建议采用：

### 3.1 独立公式

独立公式使用：

```markdown
$$
\pi_\theta(a_t \mid o_t)
$$
```

要求：

- `$$` 前后各空一行；
- 不要塞进列表句子里；
- 不要改变量名、上下标、希腊字母；
- 不要把 `\mid` 改成普通 `|`。

### 3.2 行内公式

当前 mdBook 中，行内公式优先使用：

```markdown
$\Delta x_t$
```

不建议在当前配置下使用：

```markdown
\( \Delta x_t \)
```

因为页面可能显示成：

```text
( \Delta x_t )
```

### 3.3 公式修改红线

不要做以下修改：

```text
把 $a_t$ 改成 “动作 t”
把 $o_t$ 改成 “观测 t”
把 $\pi_\theta(a_t \mid o_t)$ 改成普通文本
把 $$...$$ 拆成句子
把公式边界删除
把条件概率里的 \mid 改坏
```

### 3.4 公式索引中的公式不要放进 Markdown 表格

复杂公式不要放进 Markdown 表格单元格，尤其是包含以下符号时：

```text
\frac
\sum
\arg\min
\log
\mid
\| \cdot \|
```

这些公式在 mdBook 表格中容易被 Markdown 或 MathJax 解析坏。

推荐写法是“编号 + 独立公式 + 含义 + 掌握要求”：

```markdown
### 公式 (2.4)：连续动作下的 MSE 损失

$$
\mathcal{L}_{MSE}(\theta)
= \frac{1}{N}\sum_{i=1}^{N}\left\|f_\theta(o_i)-a_i\right\|^2
$$

- **含义**：用预测动作和专家动作之间的平方距离训练确定性策略。
- **需要掌握到什么程度**：理解 MSE 适合连续动作回归，但在多模态动作下可能产生平均动作问题。
```

简单短公式可以放在正文行内，但公式索引为了稳定，优先使用逐条独立公式块。

## 4. 章节标题与小节标题改造方法

### 4.1 章标题要表达本章知识推进路径

不建议只用口语化标题，例如：

```text
模仿学习到底在模仿什么
```

可以改成：

```text
第1章 从专家示范到策略学习：模仿学习问题的起点
```

标题最好包含两个部分：

```text
主题对象 + 本章推进方向
```

例如：

```text
第2章 Behavior Cloning：从专家样本到监督学习策略
第3章 分布偏移：为什么训练误差小不代表执行成功
第4章 DAgger：用交互数据修正策略分布
```

---

### 4.2 小节标题要专业但可读

推荐标题风格：

```text
概念名：本节回答的问题
```

例如：

```text
1.2 观测与状态：策略输入从哪里来
1.3 动作与动作空间：策略输出到哪里去
1.4 观测—动作样本：最小监督学习单元
1.5 轨迹：专家行为的时间结构
```

不建议大量使用：

```text
第一层问题：……
第二层问题：……
第三层问题：……
```

这种写法适合草稿推理，但成书目录显得不够正式。

## 5. 机械臂主线绑定方法

本书采用“视觉引导机械臂抓取与放置任务”作为统一主线。

每一章都要尽量回答：

```text
本章的数学概念，在机械臂抓取与放置任务中解决了什么问题？
```

### 5.1 讲概念时绑定机械臂

示例：

```text
观测 o_t：顶部图像、侧视图像、关节角、末端位姿、夹爪状态。
动作 a_t：末端位置增量、末端姿态增量、夹爪开合命令。
轨迹 tau：一次完整抓取与放置示范过程。
策略 pi_theta：根据当前观测输出下一步动作的规则。
```

### 5.2 讲公式时绑定机械臂

例如 Behavior Cloning 的损失函数不能只解释成“监督学习误差”，还要写成：

```text
在机械臂任务中，这个损失表示：模型看到当前图像和机械臂状态后，输出的末端控制命令要尽量接近专家遥操作命令。
```

### 5.3 讲局限时绑定机械臂

比如 BC 的局限不能只说“分布偏移”，还要说：

```text
如果模型第一步让夹爪偏了 2 厘米，下一帧相机看到的画面就不再是专家数据中的画面。模型继续按照专家数据训练出的局部规则行动，可能越走越偏，最后夹住空气或撞到物体边缘。
```

## 6. 配图规范

每章至少 3 张例图。

注意：这些图不是章节总结图，而是概念讲解图、流程图、对比图或错误模式图。

### 6.1 先盘点并优先复用已有图片

成书级改造章节时，不要默认重新生成所有图片。应先盘点仓库中本章和相关章节已经存在的图片，优先复用仍然准确、风格一致、可解释当前概念的原图。

复用前按四类处理：

```text
A 类：直接复用
图的内容仍然准确，图号和正文位置也基本匹配，只需要在正文中重新引用。

B 类：改名 / 改编号后复用
图内容可用，但文件名、图号、标题或正文位置和新版章节结构不一致，需要同步修改图注或正文引用。

C 类：作为补充图复用
新版章节已有主图，旧图可以作为局部概念补充图，例如补充“监督学习视角”“训练流程细节”“错误模式”。

D 类：弃用
图风格、编号、内容或概念表达已经和新版章节不一致，保留在仓库中，但不再正文引用。
```

第 2 章的处理示例：

```text
图2-1：使用新版 ch02_bc_training_flow.svg，强调从专家轨迹到策略部署的完整流程。
图2-2：复用旧图 图2-1_行为克隆可以看成监督学习.png，作为“监督学习视角”的补充图。
图2-3：使用新版 ch02_deterministic_vs_probabilistic_bc.svg，解释确定性 BC 与概率 BC。
图2-4：使用新版 ch02_closed_loop_failure.svg，解释闭环执行失败。
```

图片复用时必须同步检查：

- 正文图号是否连续；
- 图片文件名是否容易识别；
- 图中文字标题是否和正文图号一致；
- 图注是否说明该图在本章中的作用；
- 是否存在同一图号对应多张图片的问题。

### 6.2 推荐图类型

1. **任务总览图**
   - 用于告诉读者本章讨论的系统对象是什么。

2. **概念关系图**
   - 用于解释状态、观测、动作、轨迹、策略之间的关系。

3. **流程图**
   - 用于解释训练流程、部署流程、数据闭环。

4. **对比图**
   - 用于解释确定性策略 vs 随机策略、训练阶段 vs 执行阶段、专家分布 vs 学习策略分布。

5. **失败模式图**
   - 用于解释为什么算法会翻车。

### 6.3 第二章建议配图

第 2 章 Behavior Cloning 建议至少包含：

```text
图2-1 行为克隆训练流程：专家轨迹 → 观测—动作样本 → 策略模型 → 损失函数
图2-2 行为克隆的监督学习视角：输入 → 标签 → 损失
图2-3 确定性 BC 与概率 BC 的区别
图2-4 训练误差小但闭环执行失败：机械臂逐步偏离专家轨迹
```

## 9. 定义索引与公式索引要求

### 9.1 定义索引

每章末尾建议包含：

```markdown
## X.12 本章定义索引

| 编号 | 概念 | 一句话含义 |
|---|---|---|
| 定义 X.1 | 行为克隆 | 把专家动作作为监督学习标签训练策略 |
```

### 9.2 公式索引

每章末尾建议包含公式索引，但不要把复杂公式塞进 Markdown 表格。

推荐格式：

```markdown
## X.11 本章公式索引

### 公式 (X.1)：公式名称

$$
\hat a_t = f_\theta(o_t)
$$

- **含义**：确定性策略输入观测，输出一个预测动作。
- **需要掌握到什么程度**：理解确定性策略如何输出动作。
```

公式索引不只是列表，还要告诉读者掌握程度。

## 10. 推荐阅读与附录联动

每章末尾要有两类阅读指引。

### 10.1 建议阅读的附录条目

例如：

```text
附录 A：数学符号与公式阅读方法
附录 B：概率论最小生存包
附录 C：优化与梯度下降基础
附录 F：强化学习与序列决策基础
```

要说明为什么读：

```text
如果读者不熟悉负对数似然，需要阅读附录 B 中的条件概率和最大似然部分。
```

### 10.2 推荐阅读与深入材料

推荐阅读要说明：

```text
材料类型：基础 / 经典 / 前沿 / 工程
阅读目的：为什么读
重点看什么：读哪几节或哪类问题
对应本章：和本章哪个概念有关
```

不要只列论文名。

## 11. 成书级检查清单

每章完成后，用以下清单检查。

### 11.1 结构检查

- [ ] 章标题是否表达了本章知识推进路径？
- [ ] 小节顺序是否符合概念依赖关系？
- [ ] 是否存在“后定义的概念先出现在公式中”的问题？
- [ ] 是否和上一章、下一章形成自然衔接？

### 11.2 概念检查

- [ ] 第一次出现的重要概念是否正式定义？
- [ ] 定义是否简洁、准确、不过度展开？
- [ ] 定义之后是否有直觉解释和机械臂例子？
- [ ] 是否有本章定义索引？

### 11.3 公式检查

- [ ] 独立公式是否使用 `$$ ... $$`？
- [ ] 行内公式是否能在 mdBook 中稳定显示？
- [ ] 每个核心公式是否有读法说明？
- [ ] 每个核心公式是否有符号解释？
- [ ] 每个核心公式是否有工程含义？
- [ ] 本章公式索引是否避免把复杂公式塞进 Markdown 表格？

### 11.4 工程主线检查

- [ ] 本章是否回到机械臂抓取与放置任务？
- [ ] 是否解释了本章概念解决哪类真实失败？
- [ ] 是否说明了训练阶段和执行阶段的差别？
- [ ] 是否指出了工程实现风险？

### 11.5 图示检查

- [ ] 改造前是否先盘点已有图片？
- [ ] 是否优先复用仍然准确的原图？
- [ ] 已复用图片是否完成图号、图注、正文引用同步？
- [ ] 本章是否至少有 3 张例图？
- [ ] 图是否服务于概念理解，而不是简单总结？
- [ ] 图编号是否和正文一致？
- [ ] 图名是否清楚表达图的作用？

### 11.6 成书配套检查

- [ ] 是否有本章要点回顾？
- [ ] 是否有建议阅读的附录条目？
- [ ] 是否有思考题？
- [ ] 是否有配图清单？

## 12. 给 GPT / Codex 的改造提示词模板

后续改造章节时，可以使用以下提示词：

```text
请按照仓库中的《成书级章节改造方法.md》和《公式格式说明.md》，对第 X 章进行成书级改造。

要求：
1. 不改变全书总体目录和本章核心主题；
2. 先检查小节顺序是否符合概念依赖关系，如不合理请调整；
3. 第一次出现的重要概念必须给出“定义 X.Y”；
4. 核心公式必须使用独立公式块，并按“动机—公式—读法—符号解释—直觉—工程含义—常见误解”拆解；
5. 行内公式使用当前 mdBook 稳定显示方式，独立公式使用 $$...$$；
6. 公式索引不要把复杂公式塞进 Markdown 表格，优先使用逐条独立公式块；
7. 改造章节前先盘点已有图片，优先复用准确可用的原图，再决定是否新增图片；
8. 所有概念尽量落回“视觉引导机械臂抓取与放置任务”解释；
9. 每章至少 3 张概念讲解图，不要章节总结图；
10. 保留或补充本章公式索引、定义索引、要点回顾、附录阅读、思考题、配图清单和推荐阅读；
11. 删除内部更新说明、写作过程说明、mdBook 兼容说明等非正文内容；
12. 输出成书级正文，不要只给修改建议。
```
