## 3. 数学内容检查

第17章围绕 Transformer policy 的核心数学对象展开，重点覆盖：

- 单步策略：`πθ(a_t|o_t)`；
- 历史条件策略：`Pθ(a_t|o_{≤t},a_{<t},g)`；
- token 嵌入：`x_i = Embed(u_i)+p_i+c_i`；
- Query / Key / Value；
- self-attention 公式；
- causal mask 与 masked attention；
- history window；
- context length 与 token 数估计；
- attention 复杂度；
- sequence-to-sequence policy / action chunk；
- NLL 与 MSE 训练目标；
- action tokenization；
- 多模态条件策略；
- 泊车历史策略示例。

检查结论：

- 关键公式均配有动机说明、符号解释、直觉理解、工程案例和常见误解；
- 没有把 Transformer 误写为可自动解决物理、控制、分布偏移或安全问题的方法；
- 与第16章 Decision Transformer、第18章 VLA 的衔接自然。
