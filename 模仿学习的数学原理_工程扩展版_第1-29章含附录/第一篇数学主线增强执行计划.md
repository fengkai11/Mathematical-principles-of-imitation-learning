# 第一篇读者视角修改计划（P0-P2）

本文档用于落实第一篇“角色扮演式读者审阅”后的修改意见。

当前第一篇已经完成了一轮数学主线增强：

```text
专家轨迹数据 D_E
→ 学习策略 pi_theta(a | o)
→ BC 在专家分布 d^{pi_E}(o) 上最小化单步损失
→ 部署时策略面对自己诱导出的分布 d^{pi_theta}(o)
→ 分布不一致导致误差累积
→ DAgger 用 d^{pi_i}(o) 上的新标注修正训练分布
```

在最新读者视角下，第一篇还需要进一步强化一条更适合普通工程师读者的认知主线：

```text
普通监督学习
→ i.i.d. 或训练/测试近似同分布假设
→ Behavior Cloning 借用监督学习形式
→ 闭环动作改变未来输入
→ i.i.d. 前提失效
→ 专家分布 d^{pi_E} 与学习策略分布 d^{pi_theta} 不一致
→ 误差累积
→ DAgger 用学习策略访问状态修正训练分布
```

本次修改计划的目标不是继续堆内容，而是让读者读第一篇时更顺、更少卡顿、更容易形成完整因果链。

---

## 一、总体改造目标

第一篇最终要让读者带走的不是一组零散名词：

```text
BC、分布偏移、DAgger、MSE、NLL
```

而是一条清楚的理解路径：

```text
我熟悉监督学习
→ BC 看起来就是监督学习
→ 但策略动作会改变未来输入
→ 所以训练分布和执行分布不一致
→ 这导致误差累积
→ DAgger 的意义是把模型自己会遇到的状态补进训练集
```

第一篇的四章应形成明确接力关系：

```text
第1章：先说明模仿学习真正学的是策略，不是轨迹
第2章：策略学习最朴素的做法是 BC，看起来像监督学习
第3章：但策略闭环执行会改变未来输入，所以 BC 会分布偏移
第4章：DAgger 通过让模型自己暴露偏离状态，修正训练数据来源
```

---

## 二、改造原则

1. **不重写原章节主干**：保留第 1–4 章已有叙述、图示和阅读节奏。
2. **不继续无节制扩写**：本轮重点是增强主线、衔接、读者自检和误解澄清。
3. **从读者疑问出发**：每个新增内容都要回答一个真实阅读疑问。
4. **保持工程师友好**：公式必须翻译成工程语言，尤其是分布、期望、策略诱导分布。
5. **保持公式规范**：避免使用当前 mdBook/MathJax 不稳定写法；不随意引入复杂宏；不打乱已有核心公式编号。
6. **让每章承担一个清晰职责**：第1章定义问题，第2章建立 BC，第3章解释失败机制，第4章给出数据修正方法。

### P0-4 第3章把 `d^{pi_E}` 和 `d^{pi_theta}` 翻译成工程语言

**读者问题**：

> 这些分布符号到底对应真实系统里的什么东西？

**修改目标**：

第3章不能只给公式，还要反复把专家分布和学习策略分布翻译成机械臂、自动驾驶、泊车等工程语言。

**建议写法**：

```text
d^{pi_E}：专家正常操作时，系统经常看到的画面、姿态、相对位置和状态集合。
d^{pi_theta}：模型自己控制以后，实际会把系统带到的画面、姿态、相对位置和状态集合。
```

**机械臂例子**：

```text
专家分布里：夹爪位置合理、目标无遮挡、姿态逐步对准。
学习策略分布里：夹爪偏右、目标被遮挡、姿态异常、已经接近失败但仍可恢复。
```

**当前状态**：部分已执行，仍需加强。

### P0-5 第4章反复强调“DAgger 改变的是数据来源，不是高级 loss”

**读者问题**：

> DAgger 到底高级在哪里？是换了模型，换了 loss，还是多采了数据？

**修改目标**：

第4章必须明确：DAgger 的关键不是复杂网络，也不是神秘 loss，而是让训练数据覆盖学习策略真实访问到的分布。

**核心表达**：

```text
BC：只在专家数据上训练；
DAgger：让当前策略执行，在它实际访问到的观测上请专家补标签，并把数据聚合回训练集。
```

### P1-5 第2章增加“BC 什么时候仍然有效”

**读者问题**：

**建议新增小节**：

```text
BC 什么时候仍然有效？
```

**建议内容**：

- 任务 horizon 短；
- 专家数据覆盖足够多；
- 执行环境和训练环境差异小；
- 系统有低层控制器兜底；
- 有人工接管或在线数据补充；
- 后续结合 ACT、Diffusion Policy、DAgger、数据闭环。

**核心表达**：

> BC 是起点，不是废物。很多现代机器人策略、VLA、ACT、Diffusion Policy，本质上也离不开大规模行为数据监督学习。
