# 全书机械臂抓取主线改造方案

> 本文档用于指导《模仿学习的数学原理：工程扩展版》的后续修订。目标不是推翻现有八篇二十九章结构，而是在现有结构中加入一条清晰、稳定、可反复回到的工程主线：**视觉引导机械臂抓取与放置任务**。
>
> 这条主线的作用类似一本数学教材中的统一例子：读者每学一个新概念，都能回到同一个任务中理解它为什么出现、解决什么问题、忽略它会导致什么失败。

---

## 1. 为什么需要一条全书主线

当前书稿的优点是覆盖面完整，已经从 Behavior Cloning、DAgger、MDP、CVAE、ACT、Diffusion Policy、Flow Matching，一直写到 VLA、World Model、Preference Alignment、Sim-to-Real、OPE、C4/ADR 与数据闭环。

但这种结构也容易带来一个问题：

> 每一章都能讲清一个概念，但读者不一定能感受到这些概念是在解决同一个任务中不断暴露出来的问题。

对于一本以“数学原理”为核心的技术书，仅仅解释公式还不够。读者还需要知道：

1. 这个公式为什么会被提出；
2. 它在真实机器人任务中对应什么对象；
3. 如果不用这个概念，系统会出现什么失败；
4. 后续更复杂的方法，是如何从前面的问题自然生长出来的。

因此，全书需要一条稳定主线，把二十九章串成一个逐步深入的问题链。

---

## 2. 总体改造原则

本次改造应遵守四个原则。

### 2.1 不推翻现有目录

现有八篇、二十九章、九个附录的结构已经足够完整，不建议大规模重排。

改造方式应是：

```text
保留现有目录
在每章增加统一主线小节
重写部分开场与过渡段
在关键公式解释中回到机械臂任务
补充少量概念图与失败案例图
```

### 2.2 主线服务数学，而不是替代数学

机械臂抓取主线不是为了把本书改成“机器人项目教程”。

它的定位是：

```text
用一个真实任务解释数学对象
用工程失败现象解释数学问题
用任务递进解释算法演化
```

所以，每章仍然要保留原来的数学结构：定义、公式、目标函数、概率解释、优化解释、工程含义、常见误解。

### 2.3 主线要统一，辅助例子要降级

全书例子优先级统一为：

```text
第一主线：视觉引导机械臂抓取与放置
数学简化版：二维点机器人 / 二维夹爪接近任务
工程类比例子：自动驾驶 / 泊车 / 其他机器人任务
```

自动驾驶和泊车可以保留，但只作为辅助类比，不能再与机械臂主线并列争夺叙事中心。

### 2.4 每章都回答同一个核心问题

每章应固定追问：

> 在机械臂抓取与放置任务中，本章到底解决了什么具体困难？

例如：

- Behavior Cloning 解决的是“能不能把专家动作当标签学”；
- 分布偏移解决的是“为什么一上机就越走越偏”；
- 隐变量策略解决的是“为什么同一个观测下可能有多个合理抓法”；
- ACT 解决的是“为什么不能每一帧都孤立预测动作”；
- Diffusion Policy 解决的是“为什么动作分布不能被一个均值动作代表”；
- VLA 解决的是“为什么机器人需要理解语言条件下的任务目标”；
- OPE 解决的是“为什么不能每个策略都直接上实机试错”。

## 3. 全书统一主线任务定义

### 3.1 主线任务名称

建议统一称为：

> **视觉引导机械臂抓取与放置任务**

不要只写“机械臂抓取”，因为单纯抓取太窄，容易让读者误以为传统视觉定位与轨迹规划已经足够解决。加入“放置”以后，任务天然包含长时序、目标约束、误差累积、多模态动作、语言指令和数据闭环。

### 3.2 任务叙事

全书可以反复使用如下设定：

> 桌面上放着若干物体，例如杯子、积木、轴承套或小零件。机械臂通过顶部相机、侧视相机以及自身关节状态观察环境。人类通过遥操作示范若干条成功轨迹：接近目标、调整姿态、闭合夹爪、抬起物体、移动到目标槽位或容器上方、放下物体。模仿学习的目标，是从这些专家示范中学到一个策略，使机械臂在新位置、新物体摆放、新光照和小扰动下，也能完成抓取与放置。

这段叙事可以在第 1 章正式登场，后续章节不断回到它。

### 3.3 任务组成

#### 观测

观测记为：

\[
o_t = (I_t^{top}, I_t^{side}, q_t, e_t, g_t, h_t)
\]

其中：

- \(I_t^{top}\)：顶部相机图像；
- \(I_t^{side}\)：侧视相机图像；
- \(q_t\)：机械臂关节角或关节状态；
- \(e_t\)：末端执行器位姿；
- \(g_t\)：夹爪开合状态；
- \(h_t\)：可选的历史信息，例如过去若干帧观测或动作。

在早期章节中，不必一开始使用完整观测，可以先简化为：

\[
o_t = (I_t, q_t)
\]

再逐步扩展到多相机、多历史、多模态输入。

#### 状态

如果需要区分真实状态和观测，可以写成：

\[
s_t = (x_t^{obj}, x_t^{ee}, q_t, c_t)
\]

其中：

- \(x_t^{obj}\)：物体真实位姿；
- \(x_t^{ee}\)：末端执行器真实位姿；
- \(q_t\)：机械臂关节状态；
- \(c_t\)：接触状态，例如是否抓住物体、是否碰撞、是否放入槽位。

注意：真实状态 \(s_t\) 在很多实机任务中不可完全观测，模型通常只能看到观测 \(o_t\)。这正好可以用于解释 POMDP、历史建模、Transformer Policy 与 World Model。

#### 动作

动作可以统一写成：

\[
a_t = (\Delta x_t, \Delta r_t, u_t^g)
\]

其中：

- \(\Delta x_t\)：末端位置增量；
- \(\Delta r_t\)：末端姿态增量；
- \(u_t^g\)：夹爪控制命令，例如打开、闭合或连续开合量。

在不同章节中，可以根据需要替换为：

```text
关节速度控制：a_t = \dot q_t
关节位置控制：a_t = q_{t+1}^{cmd}
末端位姿控制：a_t = (\Delta x_t, \Delta r_t, u_t^g)
动作块控制：A_t = (a_t, a_{t+1}, ..., a_{t+H-1})
```

#### 轨迹

一条专家示范轨迹可以写成：

\[
\tau_i = (o_1^i, a_1^i, o_2^i, a_2^i, \dots, o_T^i, a_T^i)
\]

数据集为：

\[
D = \{\tau_i\}_{i=1}^{N}
\]

其中 \(N\) 是示范轨迹条数，\(T\) 是单条轨迹长度。后续讲 BC、DAgger、轨迹分布、序列损失、OPE、数据闭环时，都可以从这个定义出发。

#### 策略

模仿学习最终要学的是策略：

\[
\pi_\theta(a_t \mid o_t)
\]

或者在长时序任务中：

\[
\pi_\theta(A_t \mid o_{\le t}, a_{<t}, c)
\]

其中：

- \(A_t\)：未来一段动作块；
- \(o_{\le t}\)：当前及历史观测；
- \(a_{<t}\)：历史动作；
- \(c\)：任务条件，例如语言指令或目标描述。

这两个公式可以作为全书策略建模的主轴。

## 4. 主线任务的三层版本

为了让全书逐步展开，主线任务不应一开始就过于复杂。建议设计为三层。

### 4.1 Toy 版：二维夹爪接近任务

用于第 1–6 章。

```text
场景：二维平面上有一个物体和一个夹爪点。
观测：夹爪位置、物体位置、目标槽位位置。
动作：二维位移增量。
目标：移动到物体附近，完成抓取和放置。
```

适合解释：

- 状态、观测、动作、轨迹；
- Behavior Cloning；
- 单步损失与轨迹损失；
- 分布偏移；
- DAgger；
- MDP 与轨迹分布。

### 4.2 Standard 版：6DoF 机械臂抓取与放置

用于第 7–16 章。

```text
场景：真实或仿真的 6DoF 机械臂在桌面上抓取物体并放入槽位。
观测：RGB / RGB-D 图像、关节角、末端位姿、夹爪状态。
动作：末端位姿增量、关节速度或动作块。
目标：在扰动、遮挡、小误差下稳定完成任务。
```

适合解释：

- 确定性策略与概率策略；
- 多峰动作分布；
- 隐变量策略；
- CVAE；
- ACT；
- Diffusion Policy；
- Flow Matching；
- BC / ACT / Diffusion / Flow Matching 对比。

### 4.3 Advanced 版：语言条件下的多物体长时序操作

用于第 17–29 章。

```text
场景：桌面上有多个物体，机器人根据语言指令完成抓取、移动、放置、整理等任务。
观测：多相机图像、本体状态、历史动作、任务语言。
动作：长序列动作块或分层控制命令。
目标：泛化到新物体、新指令、新摆放方式，并通过实机数据持续改进。
```

适合解释：

- Decision Transformer；
- Transformer Policy；
- SSM / Mamba；
- VLA；
- World Model / World Action Model / JEPA；
- 快慢模型；
- Preference Alignment / DPO；
- Sim-to-Real；
- OPE；
- C4 / ADR；
- 数据闭环与后训练平台。

## 5. 每章固定新增栏目模板

建议每章新增一个统一栏目：

```markdown
## 回到机械臂抓取与放置主线
```

该栏目不需要过长，通常 500–1000 字即可。结构建议固定为四段。

### 5.1 本章概念在主线任务中对应什么

回答：本章讲的数学对象，在机械臂任务里是什么。

示例：

```text
在机械臂抓取任务中，Behavior Cloning 把每一帧专家示范整理成一个监督学习样本：输入是当前图像和机械臂状态，标签是专家此刻给出的末端运动命令和夹爪命令。
```

### 5.2 如果忽略本章问题，会出现什么失败

回答：没有这个概念，系统会怎么翻车。

示例：

```text
如果只用 BC 训练，模型在训练集附近看起来很准。但一旦末端执行器偏离专家轨迹几厘米，模型看到的观测就不再像训练数据，后续动作会越来越不可靠。
```

### 5.3 本章方法如何缓解这个失败

回答：本章方法提供了什么解决思路。

示例：

```text
DAgger 的核心价值，是把模型自己执行时遇到的状态重新交给专家标注，让训练数据覆盖“模型会犯错的地方”。
```

### 5.4 与后续章节的关系

回答：本章为什么会自然引出下一章。

示例：

```text
即使 DAgger 缓解了分布偏移，我们仍然会遇到另一个问题：同一个观测下可能有多种合理动作。于是后文需要从确定性策略进入概率策略和隐变量策略。
```

## 6. 二十九章主线嵌入地图

### 第1章 模仿学习到底在模仿什么

主线问题：机器人到底是在模仿单个动作、整条轨迹，还是一个策略函数？

建议改造：

- 用机械臂遥操作抓取与放置作为开场；
- 从一条示范轨迹引出 \(o_t\)、\(a_t\)、\(\tau\)、\(D\)、\(\pi_\theta\)；
- 降低自动驾驶和泊车例子的篇幅，将其改为辅助类比；
- 明确全书主线任务将在后续不断升级。

### 第2章 Behavior Cloning

主线问题：能不能把专家动作当成监督学习标签？

建议改造：

- 把 BC 写成机械臂观测到动作的回归问题；
- 用末端位姿增量和夹爪命令解释连续动作回归；
- 将 MSE / NLL / Gaussian policy 解释落回“专家动作标签不一定唯一”；
- 提醒读者：BC 学的是专家状态分布下的动作，不是完整闭环鲁棒性。

### 第3章 分布偏移

主线问题：为什么训练时误差很小，上机后却越走越偏？

建议改造：

- 用“夹爪偏 2cm 后看到从未见过的图像”解释 covariate shift；
- 用轨迹滚动展开解释单步误差累积；
- 补一张图：专家轨迹管道 vs 模型偏离后的状态分布；
- 强调分布偏移是模仿学习从监督学习走向序列决策的分水岭。

### 第4章 DAgger

主线问题：如何让训练数据覆盖模型自己会走到的状态？

建议改造：

- 把专家标注解释为人类在机器人执行时进行纠偏或离线重标；
- 用“模型执行到歪斜姿态，专家告诉它如何救回来”解释数据聚合；
- 讨论真实机械臂 DAgger 的成本和安全边界；
- 引出后续数据闭环平台。

### 第5章 MDP

主线问题：为什么抓取与放置不是孤立帧预测，而是序列决策？

建议改造：

- 用接近、对准、抓取、抬起、放置解释状态转移；
- 说明动作会改变下一帧观测；
- 区分状态 \(s_t\) 与观测 \(o_t\)；
- 为后文轨迹分布和 World Model 铺垫。

### 第6章 从单步损失到轨迹损失

主线问题：为什么每一步看起来都对，整条轨迹仍然可能失败？

建议改造：

- 用“每一步偏一点，最后夹爪错过目标”解释 compounding error；
- 对比单步动作误差与任务成功率；
- 引入轨迹级评价指标，例如成功率、碰撞率、放置误差；
- 铺垫 ACT 和 Diffusion Policy 的动作序列建模。

### 第7章 确定性策略与概率策略

主线问题：同一个观测下是否只有一个正确动作？

建议改造：

- 用“从左侧接近”和“从右侧接近”解释多种合理抓法；
- 说明确定性均值策略可能产生夹在中间的错误动作；
- 引出概率策略 \(\pi_\theta(a_t|o_t)\)；
- 为隐变量策略和生成式策略铺垫。

### 第8章 隐变量策略

主线问题：如何表示不同抓取风格？

建议改造：

- 用隐变量 \(z\) 表示抓取模式，例如左侧接近、右侧接近、先旋转后抓取；
- 说明 \(\pi_\theta(a|o,z)\) 与 \(\pi_\theta(a|o)\) 的区别；
- 强调隐变量不是玄学，而是对多模态动作分布的结构化表达。

### 第9章 CVAE

主线问题：训练时知道完整专家轨迹，推理时只能看当前条件，如何学多模态动作？

建议改造：

- 用 encoder 读取专家动作序列，得到潜在抓取风格；
- 用 decoder 在推理时根据图像和采样的 \(z\) 生成动作；
- 解释 KL 项时回到“不能让每条示范都记成孤立编号”；
- 强调 CVAE 是理解 ACT 的重要前置概念。

### 第10章 IRL

主线问题：专家抓取动作背后到底在优化什么偏好？

建议改造：

- 用“少碰撞、短路径、夹爪姿态稳定、放置准确”解释隐含 reward；
- 说明 IRL 不是直接抄动作，而是反推专家偏好的代价函数；
- 讨论为什么真实机械臂中 reward 设计很难。

### 第11章 GAIL

主线问题：能不能不显式写 reward，而是让判别器判断像不像专家？

建议改造：

- 将专家轨迹和策略轨迹交给判别器；
- 用“动作像不像专家示范”解释 occupancy matching；
- 说明 GAIL 对实机机器人不一定是首选，但它提供了分布匹配视角。

### 第12章 Offline Imitation Learning

主线问题：离线数据多就一定好吗？

建议改造：

- 用机械臂数据集中的成功轨迹、失败轨迹、偏置轨迹解释数据质量；
- 讨论数据覆盖、场景多样性、示范一致性；
- 引出“坑有没有录进去”的工程问题。

### 第13章 ACT

主线问题：为什么机械臂不应该每一帧孤立预测动作？

建议改造：

- 用接近、对准、闭合夹爪这一段连续动作解释 action chunk；
- 说明动作块可以减少抖动，提高短期一致性；
- 用 temporal ensemble 解释动作平滑；
- 与第 6 章轨迹损失建立明确联系。

### 第14章 Diffusion Policy

主线问题：如何从复杂多峰动作分布中生成一段可执行动作？

建议改造：

- 用“先采样一段噪声动作，再逐步去噪成抓取轨迹”解释扩散生成；
- 对比 MSE 均值回归在多模态场景中的失败；
- 强调条件输入可以是图像、本体状态和任务目标。

### 第15章 Flow Matching

主线问题：能不能用连续流场直接学习动作生成路径？

建议改造：

- 用“把噪声动作连续搬运到专家动作分布”解释 flow；
- 与 Diffusion Policy 对比：一个强调去噪过程，一个强调速度场匹配；
- 将公式解释落回动作轨迹生成。

### 第16章 BC / ACT / Diffusion Policy / Flow Matching 对比

主线问题：同一个机械臂任务，为什么不同策略模型适合不同难度？

建议改造：

- 构建统一表格：输入、输出、建模对象、适合场景、失败模式；
- 用同一个抓取与放置任务对比 BC、ACT、Diffusion、Flow Matching；
- 明确选择模型时要看任务是否长时序、是否多模态、是否需要平滑动作。

### 第17章 Decision Transformer

主线问题：能不能把机械臂决策序列当成条件序列建模？

建议改造：

- 用轨迹 token 化解释 return-to-go、observation、action；
- 讨论机器人任务中 return 条件不一定容易定义；
- 将 Decision Transformer 作为连接序列建模和策略学习的桥。

### 第18章 Transformer Policy

主线问题：为什么历史观测和动作对当前决策很重要？

建议改造：

- 用遮挡、接触状态、夹爪是否已经抓住物体解释历史依赖；
- 说明 Transformer 如何融合多帧图像、本体状态和动作历史；
- 强调 Transformer 不是魔法，而是条件建模器。

### 第19章 SSM 与 Mamba

主线问题：长时域控制能不能用更高效的序列骨架？

建议改造：

- 用长时间整理、搬运、装配任务解释长序列压力；
- 对比 Transformer 的注意力成本；
- 将 SSM / Mamba 解释为长历史压缩与状态更新机制。

### 第20章 VLA

主线问题：机器人如何根据语言指令改变操作目标？

建议改造：

- 用“抓红色杯子”“把轴承套放入左侧槽位”解释语言条件；
- 将 \(c\) 作为任务条件写入策略；
- 区分低层动作控制与高层语义理解；
- 强调 VLA 不是简单把语言拼到图像后面，而是让任务条件参与动作生成。

### 第21章 从动作模仿到世界理解

主线问题：为什么只背动作表不够？

建议改造：

- 用物体遮挡、滑动、碰撞、抓取失败解释世界理解需求；
- 说明机器人需要理解“动作会如何改变世界”；
- 从策略函数自然过渡到世界模型。

### 第22章 World Models、World Action Models 与 JEPA

主线问题：机器人能不能预测自己动作后的世界变化？

建议改造：

- 用“夹爪闭合后物体是否被抓住”“抬起后是否滑落”解释动态预测；
- 将 World Model 写成预测未来观测或隐状态；
- 将 World Action Model 写成预测动作导致的变化；
- 将 JEPA 解释为在隐空间预测可控变化。

### 第23章 快慢模型

主线问题：通用机器人是否需要高层规划和低层控制分工？

建议改造：

- 用慢系统理解任务目标和步骤；
- 用快系统输出实时动作；
- 将语言规划、视觉理解、低层控制串成层级架构；
- 为后训练和部署铺垫。

### 第24章 Preference Alignment 与 DPO

主线问题：如何用人类偏好修正机器人策略？

建议改造：

- 用两段抓取轨迹让人选择哪段更安全、更稳、更像专家；
- 解释 preference pair；
- 说明 DPO 如何从偏好中调整策略；
- 强调机器人偏好对齐不同于纯文本对齐，要面对实机安全和任务成功率。

### 第25章 从论文到实机

主线问题：策略模型如何变成可靠系统？

建议改造：

- 围绕机械臂部署链路讲数据采集、训练、仿真验证、灰度上机、异常回退；
- 强调延迟、控制频率、安全边界、失败保护；
- 让读者看到算法之外的工程约束。

### 第26章 Sim-to-Real

主线问题：仿真里成功的抓取策略，为什么实机可能失败？

建议改造：

- 用相机标定误差、摩擦差异、夹爪柔性、光照变化解释 domain gap；
- 讨论 domain randomization、系统辨识、实机微调；
- 强调仿真不是替代实机，而是降低试错成本。

### 第27章 OPE

主线问题：不上实机，如何判断一个新策略是否值得部署？

建议改造：

- 用离线轨迹评估新策略是否可能成功；
- 解释分布外动作带来的评估风险；
- 将 OPE 与安全上机流程联系起来。

### 第28章 C4 与 ADR

主线问题：如何把机器人学习系统设计成可维护工程架构？

建议改造：

- 用机械臂学习平台拆 C4：数据采集、训练、评估、部署、监控、回放；
- 用 ADR 记录关键架构决策，例如动作空间选型、模型选型、仿真器选型；
- 强调系统设计服务于持续迭代。

### 第29章 数据闭环与后训练平台

主线问题：机器人上线后如何持续变好？

建议改造：

- 用实机失败案例进入回放、标注、重训、评估、再部署流程；
- 串联 DAgger、Preference、OPE、Sim-to-Real；
- 将全书主线收束为完整数据闭环。

## 7. 统一符号建议

为了减少全书符号漂移，建议围绕主线任务统一使用以下符号。

| 符号 | 含义 | 机械臂任务解释 |
|---|---|---|
| \(o_t\) | 观测 | 图像、本体状态、夹爪状态、历史信息 |
| \(s_t\) | 状态 | 物体位姿、末端位姿、接触状态等真实状态 |
| \(a_t\) | 单步动作 | 末端位姿增量、关节速度、夹爪命令 |
| \(A_t\) | 动作块 | 从当前时刻开始的一段未来动作 |
| \(\tau\) | 轨迹 | 一次完整抓取与放置示范 |
| \(D\) | 数据集 | 多条专家示范轨迹集合 |
| \(\pi_\theta\) | 策略 | 从观测到动作或动作分布的模型 |
| \(z\) | 隐变量 | 抓取风格、接近方向、动作模式 |
| \(c\) | 条件 | 语言指令、目标物体、目标槽位 |
| \(p(\tau)\) | 轨迹分布 | 策略与环境共同诱导出的轨迹概率 |
| \(J(\pi)\) | 策略目标 | 成功率、轨迹质量或期望回报 |

建议后续章节公式尽量使用这些符号，避免同一概念反复换符号。

## 9. 写作风格改造要求

### 9.1 每个公式都尽量落回主线对象

例如讲：

\[
\mathcal{L}_{BC}(\theta) = - \sum_{(o_t,a_t) \in D} \log \pi_\theta(a_t|o_t)
\]

不要只解释成“最大化专家动作似然”，而要补一句：

```text
在机械臂任务中，这个目标函数的意思是：当模型看到与专家示范相同的图像和机械臂状态时，它应当给专家当时给出的末端运动和夹爪控制命令更高概率。
```

### 9.2 每个抽象概念都配一个失败现象

例如：

```text
分布偏移 → 夹爪偏离专家轨迹后越修越歪
多模态动作 → 左抓和右抓被平均成撞物体的动作
轨迹损失 → 每一步误差很小但最后放不进槽位
Sim-to-Real → 仿真摩擦太理想，实机一夹就滑
OPE → 离线数据没有覆盖新策略动作，评估结果虚高
```

### 9.3 保留口语化，但减少例子漂移

当前书稿的口语化风格是优点，应保留。但后续修订时需要减少“这一段讲杯子、下一段讲泊车、再下一段讲自动驾驶”的漂移。

建议写法：

```text
主线先讲机械臂。
如果需要类比，再用一句话对照自动驾驶或泊车。
类比结束后立刻回到机械臂。
```

## 11. 成书级验收标准

修订完成后，可以用以下标准检查。

### 11.1 主线一致性

- 读者是否能明确知道全书主线任务是什么？
- 第 1 章是否正式引入这条主线？
- 每章是否至少有一处回到机械臂抓取与放置任务？
- 自动驾驶、泊车等例子是否只是辅助，而不是抢走主线？

### 11.2 数学解释一致性

- 每个核心公式是否解释了“公式想解决什么问题”？
- 每个符号是否能落回机械臂任务对象？
- 是否避免了公式突然空降？
- 是否能让只有基础数学背景的算法工程师读懂？

### 11.3 工程失败闭环

- 每章是否说明了如果忽略本章问题，机器人会如何失败？
- 是否能从失败现象自然引出本章方法？
- 是否能从本章方法自然过渡到后续章节？

### 11.4 章节递进感

读者读完后应能感受到如下递进：

```text
专家示范是什么
→ 把示范当监督学习为什么可行
→ 为什么 BC 会分布偏移
→ 为什么需要交互纠偏
→ 为什么任务本质是序列决策
→ 为什么动作不是唯一答案
→ 为什么需要隐变量和生成式策略
→ 为什么需要动作块和长序列建模
→ 为什么语言、世界模型和快慢系统会进入机器人
→ 为什么实机部署需要评估、架构和数据闭环
```

如果读者能跟着这条链条走完整本书，说明主线改造成功。

## 12. 推荐最终效果

改造后的本书不应只是一本“模仿学习主题大全”，而应成为一本围绕真实机器人任务逐步展开的数学教程。

理想阅读体验是：

> 读者从一台机械臂抓取与放置任务出发，先理解专家示范和策略函数，再看到 BC 的朴素性和分布偏移的必然性，然后逐步理解 DAgger、MDP、轨迹分布、概率策略、隐变量策略、ACT、Diffusion Policy、Flow Matching、VLA、World Model、OPE 与数据闭环。到最后，读者不仅知道每个算法的公式，还知道它们分别是在机器人系统中解决哪一种具体失败。

这就是“机械臂抓取主线”的核心价值。
