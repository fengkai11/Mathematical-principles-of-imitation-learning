# 第23章：快慢模型：通用机器人为什么需要双系统架构

> **新版布局位置**：本章属于 **第六篇：世界模型、快慢系统与认知架构**。本章是第20章 VLA、第21章世界理解、第22章 World Models / WAM / JEPA 与第25章实机部署之间的架构桥梁。


> **本章一句话导读**：本章用快慢模型组织全书现代架构，解释慢模型负责理解规划，快模型负责低延迟动作控制。

---

## 0. 本章要解决的问题

前面几章已经把现代机器人策略的主要技术拼图展开了：

- 第13章 ACT 说明了为什么机器人可以一次预测一小段动作；
- 第14章 Diffusion Policy 说明了为什么复杂动作分布需要生成式策略；
- 第15章 Flow Matching 说明了动作生成也可以写成连续流场；
- 第18章 Transformer Policy 和第20章 VLA 说明了大模型如何接入机器人策略；
- 第22章 World Models / WAM / JEPA 说明了机器人不能只看当前输入，还要预测未来世界。

但是，这里还缺一个非常重要的架构问题：

> 一个通用机器人系统，能不能只靠一个大模型从图像和语言直接输出所有高频控制动作？

直觉上，这很诱人：

<div class="math">\[
(o_t, q) \longrightarrow a_t
\tag{23.1}
\]</div>

其中 <span class="math">\\(o\_t\\)</span> 是当前观测，<span class="math">\\(q\\)</span> 是语言指令，<span class="math">\\(a\_t\\)</span> 是动作。

但在真实机器人系统中，这种“一口气端到端”的方案很快会遇到三个问题：

1. **慢推理问题**：大 VLM / VLA 擅长语义理解，但推理延迟通常较高；
2. **高频控制问题**：机械臂、灵巧手、双臂协调需要稳定、低延迟、连续动作；
3. **层级抽象问题**：任务理解、子目标规划、世界预测和动作控制不应该都压在同一个时间尺度上。

快慢模型要解决的正是这个问题。

本章的核心观点是：

> 通用机器人策略很可能不是一个单一频率、单一模块的端到端模型，而是由“慢模型”和“快模型”组成的双系统架构。

## 1. 一个直观场景：让机器人收拾桌面

假设机器人接收到指令：

> 把桌子上的杯子放到水槽旁边，顺手把抹布叠好。

这个任务至少包含两类能力。

第一类是慢能力：

- 理解“杯子”“水槽”“抹布”分别是什么；
- 判断哪个物体先处理；
- 规划大致步骤；
- 发现杯子里可能有水；
- 预测抓取后物体会怎么运动；
- 决定失败时是否重试或换策略。

第二类是快能力：

- 手臂运动要平滑；
- 夹爪闭合要及时；
- 接触力不能过大；
- 抓杯子时要实时补偿偏差；
- 抹布折叠时要连续控制末端轨迹。

如果让同一个大模型在每个控制周期都重新理解语言、看图、推理、规划、输出关节命令，系统会非常慢；如果只用一个高频动作模型，它又可能不知道“为什么要这么做”。

因此更合理的结构是：

```text
慢模型：理解任务、预测世界、给出子目标或动作意图
快模型：在短时间内把子目标转成连续动作并闭环执行
```

## 2. 快慢模型的数学抽象

我们把机器人系统拆成两个时间尺度。

慢时间尺度记为 <span class="math">\\(k\\)</span>，快时间尺度记为 <span class="math">\\(t\\)</span>。通常一个慢周期会覆盖多个快控制周期：

<div class="math">\[
t \in \{kM, kM+1, \dots, (k+1)M-1\}
\tag{23.2}
\]</div>

其中 <span class="math">\\(M\\)</span> 表示一个慢模型决策周期内包含多少个快控制步。

### 2.1 慢模型：从观测历史和指令生成抽象意图

慢模型可以写成：

<div class="math">\[
z_k = g_\phi(o_{\le kM}, q)
\tag{23.3}
\]</div>

其中：

- <span class="math">\\(o\_{\le kM}\\)</span>：到当前慢周期为止的观测历史；
- <span class="math">\\(q\\)</span>：语言指令或任务描述；
- <span class="math">\\(g\_\phi\\)</span>：慢模型，可以是 VLM、VLA、高层 planner 或 world model；
- <span class="math">\\(z\_k\\)</span>：慢模型输出的抽象表示，可以是子目标、计划、latent instruction、goal image 或动作意图。

这个式子的含义不是“慢模型直接控制电机”，而是：

> 慢模型负责把复杂任务压缩成一段可执行的高层条件。

### 2.2 快模型：在高层条件下输出连续动作

快模型可以写成：

<div class="math">\[
a_t \sim \pi_\theta(a_t \mid o_{t-L:t}, z_k)
\tag{23.4}
\]</div>

其中：

- <span class="math">\\(o\_{t-L:t}\\)</span>：最近一段短历史观测；
- <span class="math">\\(z\_k\\)</span>：慢模型给出的当前高层条件；
- <span class="math">\\(\pi\_\theta\\)</span>：快动作策略；
- <span class="math">\\(a\_t\\)</span>：快控制周期下的动作。

如果快模型一次输出动作块，可以写成：

<div class="math">\[
A_t = a_{t:t+H-1} \sim \pi_\theta(A_t \mid o_{t-L:t}, z_k)
\tag{23.5}
\]</div>

这就把第13章 ACT、第14章 Diffusion Policy、第15章 Flow Matching 都纳入同一个框架：

- ACT：<span class="math">\\(\pi\_\theta\\)</span> 通过 action chunk 直接输出 <span class="math">\\(A\_t\\)</span>；
- Diffusion Policy：<span class="math">\\(\pi\_\theta\\)</span> 通过反向去噪生成 <span class="math">\\(A\_t\\)</span>；
- Flow Matching：<span class="math">\\(\pi\_\theta\\)</span> 通过速度场积分生成 <span class="math">\\(A\_t\\)</span>。

## 3. 频率约束：为什么必须区分快和慢？

快慢模型不是为了好听，而是由控制频率约束逼出来的。

假设快控制周期是：

<div class="math">\[
\Delta t_{\text{fast}} = \frac{1}{f_{\text{fast}}}
\tag{23.6}
\]</div>

慢模型推理耗时为 <span class="math">\\(T\_{\text{slow}}\\)</span>，快模型推理耗时为 <span class="math">\\(T\_{\text{fast}}\\)</span>。

如果一个模型要直接参与高频控制，它至少需要满足：

<div class="math">\[
T_{\text{fast}} < \Delta t_{\text{fast}}
\tag{23.7}
\]</div>

但大模型推理通常更适合低频运行：

<div class="math">\[
T_{\text{slow}} \gg \Delta t_{\text{fast}}
\tag{23.8}
\]</div>

所以系统不能等待慢模型每一步都输出动作，而应该让慢模型低频更新条件 <span class="math">\\(z\_k\\)</span>，让快模型高频执行。

这就是快慢模型的工程根源：

> 慢模型负责“想清楚”，快模型负责“做得稳”。

## 4. 异步执行：慢模型不阻塞快控制

真实系统里，慢模型和快模型通常不是严格同步执行，而是异步通信。

可以用一个条件缓存来表示：

<div class="math">\[
\bar z_t = z_{\kappa(t)}
\tag{23.9}
\]</div>

其中：

- <span class="math">\\(\kappa(t)\\)</span>：在快时间 <span class="math">\\(t\\)</span> 时，最近一次已经完成的慢模型决策编号；
- <span class="math">\\(\bar z\_t\\)</span>：快模型当前可用的高层条件。

于是快模型实际执行的是：

<div class="math">\[
a_t \sim \pi_\theta(a_t \mid o_{t-L:t}, \bar z_t)
\tag{23.10}
\]</div>

这说明：

- 慢模型可以晚一点更新；
- 快模型不能停下来等；
- 如果慢模型还没返回，快模型就使用最近一次有效条件继续执行；
- 当条件过期或风险上升时，系统应该触发 fallback。

## 5. 快模型和慢模型分别对应前文哪些方法？

### 5.1 慢模型候选

慢模型可以来自：

- **VLA / VLM**：理解图像、语言和任务语义；
- **World Model / WAM / JEPA**：预测未来隐状态或物理后果；
- **高层 planner**：拆分任务、选择子目标；
- **价值模型 / 偏好模型**：判断哪种计划更安全、更符合人类偏好。

数学上，慢模型经常处理的是：

<div class="math">\[
p(z_{k+1}\mid z_k, u_k)
\tag{23.11}
\]</div>

或者：

<div class="math">\[
\hat y_k = h_\phi(o_{\le kM}, q)
\tag{23.12}
\]</div>

它关注的是抽象状态、目标、计划和未来预测。

### 5.2 快模型候选

快模型可以来自：

- **BC**：规则任务中直接回归动作；
- **ACT**：输出短 horizon 的动作块；
- **Diffusion Policy**：生成复杂多峰动作块；
- **Flow Matching**：用连续流场生成动作；
- **SSM / Mamba policy**：用线性长序列骨架维持高频历史状态。

数学上，快模型关注的是：

<div class="math">\[
\pi_\theta(A_t \mid o_{t-L:t}, z_k)
\tag{23.13}
\]</div>

它的重点不是“想清楚任务”，而是“把动作做稳、做快、做连续”。

## 6. 快慢模型与世界模型的关系

第22章讲的世界模型可以自然放到慢模型中。

假设慢模型内部维护一个 latent state：

<div class="math">\[
h_k = e_\phi(o_{\le kM})
\tag{23.14}
\]</div>

并预测候选高层动作 <span class="math">\\(u\_k\\)</span> 的后果：

<div class="math">\[
\hat h_{k+1} = F_\psi(h_k, u_k)
\tag{23.15}
\]</div>

那么慢模型可以通过比较不同候选计划的预测结果来选择更好的子目标：

<div class="math">\[
u_k^* = \arg\max_{u_k} J(\hat h_{k+1}, q)
\tag{23.16}
\]</div>

这里的 <span class="math">\\(J\\)</span> 可以表示任务进度、安全性、偏好分数或价值估计。

选出的 <span class="math">\\(u\_k^*\\)</span> 不一定直接是关节动作，它更可能转化为快模型条件：

<div class="math">\[
z_k = r_\eta(u_k^*, \hat h_{k+1}, q)
\tag{23.17}
\]</div>

然后快模型负责执行。

## 7. 快慢模型与 DPO / 后训练的关系

第24章的 Preference Alignment 与 DPO 可以分别作用在快慢两层。

慢模型偏好数据可以是：

<div class="math">\[
(z_k^{+}, z_k^{-})
\tag{23.18}
\]</div>

表示同一个场景下，人类更偏好某个高层计划而不是另一个高层计划。

快模型偏好数据可以是：

<div class="math">\[
(A_t^{+}, A_t^{-})
\tag{23.19}
\]</div>

表示同一个条件下，人类更偏好某段安全、平滑、成功的动作轨迹，而不是失败、碰撞或不稳定的轨迹。

因此，机器人后训练并不只是在“语言输出”上做偏好对齐，而是可以扩展为：

<div class="math">\[
\mathcal{D}_{\text{pref}}
=
\{(c_i, y_i^+, y_i^-)\}_{i=1}^{N}
\tag{23.20}
\]</div>

其中 <span class="math">\\(c\_i\\)</span> 可以是场景条件，<span class="math">\\(y\_i\\)</span> 可以是高层计划，也可以是低层动作轨迹。

## 8. 工程实现中的典型系统形态

一个实际快慢系统可以写成如下结构：

```text
语言指令 + 图像观测
        ↓
慢模型：VLM / VLA / World Model
        ↓
子目标 / latent intention / goal representation
        ↓
快模型：ACT / Diffusion / Flow / SSM Policy
        ↓
连续动作 / action chunk
        ↓
安全约束 / fallback / controller
        ↓
真实机器人
```

这套结构有一个关键优点：

> 慢模型可以越来越聪明，快模型可以越来越稳，二者不必在同一个频率和同一个模型容量上互相拖累。

## 9. 常见误解与适用边界

### 误解 1：快慢模型就是两个模型随便串起来

不是。快慢模型的关键不是数量，而是**时间尺度分工**。

如果两个模型都低频推理，仍然无法解决高频控制问题；如果两个模型都只输出动作，也没有形成抽象层级。

### 误解 2：慢模型一定是大语言模型

不一定。慢模型可以是 VLM、VLA、世界模型、规划器、价值模型，也可以是传统任务规划模块。

### 误解 3：快模型一定是小模型

也不一定。快模型的核心要求是低延迟和稳定控制，而不是参数量一定小。一个被蒸馏、裁剪、量化或部署在高性能边缘硬件上的生成式策略，也可以作为快模型。

### 误解 4：快慢模型能自动解决安全问题

不能。它只是提供架构分工。安全仍然需要第25章中的限幅、碰撞检测、fallback、异常检测和人工接管机制。

## 10. 回到统一例子

### 10.1 二维点机器人

慢模型可以决定：

- 先绕过障碍物左侧；
- 到达中间 waypoint；
- 再靠近目标点。

快模型负责：

- 在每个短周期输出速度；
- 让轨迹平滑；
- 避免过冲和振荡。

### 10.2 机械臂抓取

慢模型可以决定：

- 先从哪一侧接近物体；
- 抓取哪个部位；
- 是否需要先推开遮挡物；
- 抓取失败后是否换抓取姿态。

快模型负责：

- 末端轨迹跟踪；
- 夹爪闭合时机；
- 接触过程中的微调；
- 把动作块稳定执行到控制器层。

## 13. 本章小结

快慢模型不是一个孤立算法，而是一种组织机器人智能系统的架构原则。

它把全书前面的内容重新串成一条系统链路：

```text
慢模型：VLA / World Model / JEPA / planner
    ↓
高层条件：子目标 / latent intention / goal representation
    ↓
快模型：ACT / Diffusion Policy / Flow Matching / SSM-Mamba
    ↓
动作执行：连续控制 / 安全约束 / fallback
    ↓
后训练：Preference Alignment / DPO / 接管纠偏
```

这也是为什么本章放在 World Models / WAM / JEPA 之后、Preference Alignment / DPO 之前：
它一方面承接“机器人要理解和预测世界”，另一方面为“如何后训练和部署一个真正可用的机器人策略系统”搭桥。
