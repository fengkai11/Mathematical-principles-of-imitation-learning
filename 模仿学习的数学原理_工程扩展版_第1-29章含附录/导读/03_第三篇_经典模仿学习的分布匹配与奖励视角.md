# 第三篇导读：经典模仿学习的分布匹配与奖励视角

> **本篇一句话导读**：第二篇已经说明策略会诱导轨迹分布，单步动作损失不能完全代表闭环任务成功。第三篇继续追问：如果只模仿单步动作不够，我们应该在什么层面理解专家行为？答案会从 reward、轨迹回报、occupancy measure，一直推进到离线数据覆盖与 OOD 风险。

---

## 1. 为什么第二篇之后需要第三篇

第二篇解决了一个关键问题：模仿学习不是普通监督学习的简单变体。

在普通监督学习里，模型预测错一张图片，不会改变下一张图片的分布。但在模仿学习和机器人控制中，策略输出的动作会改变环境，环境再返回新的状态或观测。于是策略不是被动预测器，而是会主动诱导自己的未来输入。

第二篇已经建立了这条主线：

```text
状态与动作
→ 状态转移
→ 策略闭环执行
→ 轨迹分布
→ 轨迹级目标
→ 单步 imitation loss 与长期任务成功之间的差距
```

但到这里还没有回答一个更高层的问题：

> 如果单步动作拟合不够，我们到底应该匹配什么？

有三种自然的回答。

第一种回答是：也许专家行为背后有一个隐藏的目标函数。专家不是随机做动作，而是在隐式优化安全、效率、平顺、成功率等偏好。于是第10章进入 IRL，追问专家到底在优化什么。

第二种回答是：也许我们不必显式恢复 reward，只要让学习策略的整体行为分布像专家就行。于是第11章进入 GAIL，用判别器和 occupancy measure 讨论专家与策略的分布匹配。

第三种回答是：真实机器人往往不能让当前策略随便试错和 rollout。如果训练只能依赖已有数据，那么问题就变成数据是否覆盖了部署时可能遇到的状态、动作、失败和恢复。于是第12章进入 Offline Imitation Learning。

因此，第三篇不是三个经典方法的并列介绍，而是第二篇之后的一次主线升级：

```text
从“单步动作是否像专家”
→ 到“整条轨迹是否体现专家偏好”
→ 到“策略访问的状态—动作分布是否像专家”
→ 到“离线数据是否覆盖策略部署时的真实风险”
```

## 2. 本篇包含章节

- 第10章：IRL：专家到底在优化什么
- 第11章：GAIL：从判别器里偷一个奖励函数
- 第12章：Offline Imitation Learning：离线数据不是越多越好，是坑有没有录进去

这三章的关系可以理解为：

```text
第10章 IRL：从专家轨迹反推 reward，解释专家为什么这样做。
第11章 GAIL：绕开显式 reward，直接匹配专家与策略的 occupancy measure。
第12章 Offline IL：当不能在线 rollout 时，重新审视离线数据覆盖、support mismatch 和 OOD action。
```

## 3. 第三篇的数学主线

本篇的数学主线可以写成：

```text
第二篇：策略诱导轨迹分布 p_pi(tau)
→ 第10章：引入 reward r(s,a) 和轨迹回报 R(tau)
→ IRL：从专家轨迹反推能解释专家行为的 reward
→ reward ambiguity：同一专家行为可能被多个 reward 解释
→ maximum entropy IRL：用概率分布描述专家轨迹偏好
→ 第11章：从 reward inference 转向 occupancy measure matching
→ GAIL：用判别器比较专家和策略的 rho_pi(s,a)
→ 第12章：当 rollout 受限时，离线数据 support 成为核心约束
→ OOD action：动作出圈会继续改变后续状态分布
→ 第四篇：在离线数据现实下，用更强策略模型表达动作块和多模态动作分布
```

这条主线里，每一章都在回答上一章留下的问题：

| 章节 | 承接的问题 | 本章推进 | 留给下一章的问题 |
|---|---|---|---|
| 第10章 IRL | 轨迹级目标已经出现，但专家为什么选择这些轨迹还没有解释 | 引入 reward、轨迹回报、特征期望和 reward ambiguity | 如果 reward 不唯一且难恢复，能不能直接匹配行为分布？ |
| 第11章 GAIL | IRL 显式恢复 reward 存在歧义和求解困难 | 引入 occupancy measure 和对抗分布匹配 | 如果 GAIL 需要 rollout，真实机器人不能随便试错怎么办？ |
| 第12章 Offline IL | 在线 rollout 有安全、成本和设备风险 | 引入 behavior policy、support mismatch、OOD action 和 recovery data | 在固定数据上，如何用更强策略模型学习动作序列和多模态动作？ |

## 4. 本篇新增数学对象

第三篇会新增或强化以下数学对象。读者不需要一开始全部掌握，但要知道它们为什么出现。

| 数学对象 | 符号 | 一句话含义 | 工程对应物 | 后续作用 |
|---|---|---|---|---|
| 奖励函数 | $r(s,a)$ | 描述某个状态—动作好坏的标量函数 | 安全、平顺、效率、成功率、碰撞风险等偏好 | IRL、RL、后训练与策略评估 |
| 轨迹回报 | $R(\tau)$ | 一整条轨迹累计得到的 reward | 一次抓取、一次装配或一次泊车全过程质量 | 连接轨迹级目标与专家偏好 |
| 特征函数 | $f(s,a)$ | 从状态—动作中提取可比较的任务特征 | 距离、速度、碰撞余量、姿态误差、接触状态 | 线性 reward 与特征匹配 |
| 特征期望 | $\mu(\pi)$ | 策略执行后访问到的特征平均统计 | 一批轨迹中的安全距离、平顺性、成功状态统计 | IRL 中比较专家和学习策略 |
| 最大熵轨迹分布 | $p_w(\tau)$ | 用 reward 给轨迹分配概率，同时保留多种可能行为 | 专家不是只有一条唯一正确轨迹 | 最大熵 IRL 与生成式视角 |
| 占用测度 | $\rho_\pi(s,a)$ | 策略闭环执行时访问状态并执行动作的频率 | 模型实际会走到哪里、做什么 | GAIL、分布匹配和后续 OPE |
| 判别器 | $D_\omega(s,a)$ | 判断状态—动作样本更像专家还是当前策略 | 行为分布的“质检员” | GAIL 中产生隐式 reward |
| 行为策略 | $\beta(a \mid s)$ | 产生离线数据的策略 | 遥操作员、旧模型、规则系统或混合数据源 | Offline IL / Offline RL 数据来源分析 |
| 数据支撑集 | $support(\mathcal{D})$ | 离线数据实际覆盖到的状态—动作区域 | 训练集中见过的正常、边界、失败和恢复场景 | 判断 OOD 风险 |
| OOD 动作 | OOD action | 超出数据覆盖范围的动作 | 夹爪乱动、车速过大、末端姿态异常、恢复动作缺失 | 离线策略部署风险分析 |

这些对象共同完成一个转变：

```text
从监督学习里的样本点
→ 到闭环控制里的轨迹
→ 到专家偏好的 reward
→ 到整体行为的 occupancy measure
→ 到离线数据可覆盖与不可覆盖的区域
```

## 5. 本篇公式主线

本篇的公式不是为了增加形式感，而是为了让读者看清三件事：

1. reward 如何把单步状态—动作关系提升为轨迹级偏好；
2. occupancy measure 如何描述策略闭环执行后的整体行为分布；
3. offline dataset 为什么不能自动代表部署时的全部状态—动作空间。

建议读者把本篇公式按下面关系理解：

```text
轨迹分布 p_pi(tau)
→ reward r(s,a)
→ 轨迹回报 R(tau)
→ 线性 reward r_w(s,a)
→ 特征期望 mu(pi)
→ 最大熵轨迹分布 p_w(tau)
→ occupancy measure rho_pi(s,a)
→ GAIL 的 min-max 分布匹配目标
→ 离线数据 D 与 behavior policy beta
→ support(D) 与 OOD action
```

读本篇时，不要只问“这个公式怎么背”，而要问：

```text
这个公式把模仿学习推进到了哪个层面？
它是在解释专家偏好、匹配行为分布，还是分析离线数据风险？
它解决了上一章什么问题？
它又暴露了什么新的局限？
```

## 6. 与统一工程主线的关系

本书采用“视觉引导机械臂抓取与放置任务”作为统一工程主线。第三篇中的抽象对象可以落回这个任务中理解。

### 6.1 IRL 对应什么工程问题

在机械臂抓取与放置中，专家轨迹不只是末端位姿序列。它背后可能隐含很多偏好：

- 接近工件时速度要慢；
- 夹爪闭合前姿态要对齐；
- 放入治具时宁愿多做微调，也不要硬插；
- 避免碰撞、划伤、卡死；
- 成功率、节拍、冲击力之间要做权衡。

IRL 试图把这些偏好写成 reward。但读者要记住：IRL 学到的 reward 更像一种行为解释，不一定是专家内心真实目标。

### 6.2 GAIL 对应什么工程问题

GAIL 不要求显式写出“安全 + 平顺 + 成功率”的 reward，而是问：

> 当前策略产生的状态—动作分布，整体上像不像专家？

在机械臂任务中，这意味着不仅要看某一步末端增量是否接近专家，还要看策略是否经常访问相似的接近姿态、对齐阶段、夹取阶段、放置阶段和恢复阶段。

### 6.3 Offline IL 对应什么工程问题

真实机器人往往不能像仿真环境一样随便试错。夹具会磨损，工件会报废，治具会变形，现场也不允许模型频繁探索危险动作。

因此，如果训练只能依赖离线数据，核心问题就不是：

```text
数据有多少条？
```

而是：

```text
数据有没有覆盖正常轨迹？
数据有没有覆盖边界状态？
数据有没有覆盖失败状态？
数据有没有覆盖从失败中恢复的动作？
数据有没有覆盖部署时策略可能自己走进去的状态？
```

这正是第12章要强调的工程判断。

## 7. 读完本篇，你应该能判断什么

读完第三篇后，读者应该能够形成以下判断。

| 工程现象 | 应该想到的数学问题 |
|---|---|
| BC 模型单步误差不大，但整体轨迹不像专家 | 可能不是单步动作问题，而是轨迹级偏好或行为分布问题 |
| 专家有多种合理操作方式，不能只用一条标准轨迹解释 | reward ambiguity 或多模态专家行为需要被考虑 |
| 一个 reward 看起来能解释专家轨迹，但换一个 reward 也能解释 | IRL 的 reward 不唯一，不能把学到的 reward 当作绝对真理 |
| GAIL 训练时判别器很快分清专家和策略 | 策略诱导的 occupancy measure 与专家差异很大 |
| GAIL 理论上漂亮，但实机上很难用 | 当前策略 rollout 有安全、成本和设备损耗问题 |
| 离线数据很多，但上线后模型仍然进坑 | 数据量不等于 support 覆盖，可能缺少边界、失败和恢复数据 |
| 模型一旦输出奇怪动作，后面越来越不可控 | OOD action 通过环境转移诱导了新的 OOD state |
| 成功轨迹很多，失败恢复轨迹很少 | 模型可能只会顺利执行，不会从偏差中恢复 |
| 想上 ACT / Diffusion Policy | 先确认数据覆盖和任务定义，否则强模型只是更强地拟合数据偏差 |

## 9. 本篇常见误解

| 常见误解 | 正确认识 |
|---|---|
| IRL 就是从专家数据里学出真实 reward | IRL 学到的是能解释行为的 reward，reward 本身通常不唯一 |
| GAIL 就是把 GAN 套到轨迹上 | GAIL 的关键是 occupancy measure matching，不只是 GAN 类比 |
| 判别器输出就是最终可上线的奖励函数 | 判别器反馈可以作为隐式 reward，但直接上线需要非常谨慎 |
| 离线数据越多，模型越安全 | 关键不是总量，而是 support 是否覆盖部署风险 |
| 只要有成功轨迹，就能学会成功策略 | 缺少失败、边界和恢复轨迹时，模型可能不会处理偏差状态 |
| OOD 只是输入分布问题 | 在闭环控制中，OOD action 会继续改变未来状态分布 |
| 上更强模型就能解决数据覆盖问题 | ACT、Diffusion Policy、Flow Matching 提高表达能力，但不能凭空补齐没采到的数据 |

## 10. 第三篇推荐阅读与查资料清单

本清单不是为了堆论文，而是帮助读者带着问题去查资料。

### 10.1 IRL 与 MaxEnt IRL

| 推荐主题 | 阅读目的 | 查资料关键词 |
|---|---|---|
| 逆强化学习基础 | 理解“从专家行为反推 reward”的基本问题设定 | inverse reinforcement learning, apprenticeship learning |
| reward ambiguity | 理解为什么专家轨迹不能唯一决定 reward | reward ambiguity, reward identifiability, reward shaping |
| maximum entropy IRL | 理解为什么用概率方式建模专家轨迹，以及为什么会出现 partition function | maximum entropy IRL, trajectory likelihood, partition function |
| feature expectation matching | 理解线性 reward 下专家和策略如何通过特征统计比较 | feature expectation, linear reward IRL, apprenticeship learning |

阅读时重点追问：

```text
这篇资料是在学 reward，还是在匹配行为统计？
它是否承认 reward 不唯一？
它如何估计模型分布下的特征期望？
```

### 10.2 GAIL 与 occupancy measure matching

| 推荐主题 | 阅读目的 | 查资料关键词 |
|---|---|---|
| GAIL 原始思想 | 理解为什么可以绕开显式 reward，直接匹配 occupancy measure | generative adversarial imitation learning, GAIL |
| occupancy measure | 理解策略闭环执行后在状态—动作空间留下的分布 | occupancy measure, discounted visitation distribution |
| 判别器与密度比 | 理解最优判别器为什么反映专家分布和策略分布的相对大小 | density ratio estimation, discriminator optimal form |
| GAIL 工程实现 | 理解 rollout、判别器训练、策略优化之间的循环 | GAIL implementation, policy rollout, discriminator reward |

阅读时重点追问：

```text
判别器输出到底代表专家概率还是策略概率？
隐式 reward 是如何从判别器输出构造出来的？
算法是否需要环境交互或仿真 rollout？
```

### 10.3 Offline IL、数据覆盖与恢复数据

| 推荐主题 | 阅读目的 | 查资料关键词 |
|---|---|---|
| 离线模仿学习 | 理解固定数据集训练策略时的基本限制 | offline imitation learning, batch imitation learning |
| behavior policy | 理解数据由谁产生，以及采集策略如何决定数据分布 | behavior policy, data collection policy |
| support mismatch | 理解训练数据覆盖和部署策略访问区域不一致的风险 | support mismatch, distribution shift, extrapolation error |
| OOD action | 理解动作出圈如何通过状态转移放大闭环风险 | out-of-distribution action, closed-loop distribution shift |
| recovery data | 理解失败、接管、纠偏数据为什么对闭环鲁棒性重要 | recovery data, intervention data, corrective demonstrations |

阅读时重点追问：

```text
这份数据覆盖了哪些正常状态？
覆盖了哪些边界状态？
有没有失败早期和恢复动作？
采集策略 beta 是专家、人类混合、旧模型，还是规则系统？
```

### 10.4 面向工程落地的查资料路径

如果读者正在做机器人、自动驾驶或泊车项目，建议按下面顺序查资料：

```text
第一步：查 imitation learning distribution shift，理解为什么闭环分布会变。
第二步：查 IRL reward ambiguity，理解为什么 reward 不能唯一恢复。
第三步：查 occupancy measure GAIL，理解分布匹配对象是什么。
第四步：查 offline imitation learning support mismatch，理解固定数据集的风险。
第五步：查 corrective demonstrations / recovery data，理解如何补边界与恢复样本。
第六步：再进入 ACT / Diffusion Policy，理解现代策略模型如何表达动作序列。
```

## 12. 本篇如何引出第四篇

第三篇最后会把问题推到一个非常现实的位置：

```text
我们已经知道：
- 单步动作拟合不够；
- reward 学习存在歧义；
- occupancy measure matching 需要 rollout；
- 离线数据存在 support mismatch 和 OOD 风险。

那么，在真实机器人项目中，
我们通常只能拿着有限的离线数据，训练一个可以部署的策略模型。

这时下一个问题就是：
策略模型本身应该怎样表达动作？
```

如果策略每次只输出一步动作，它可能短视、抖动，也很难表达一段连续操作意图。

如果专家动作本身是多模态的，简单 MSE 又容易把多个合理动作平均成一个不合理动作。

如果任务需要连续几步协调，例如靠近、对齐、夹取、放置，那么策略最好能建模动作序列，而不是只盯着下一步。

这就自然引出第四篇：现代机器人策略模型。

```text
第13章 ACT：用动作块缓解单步预测短视问题。
第14章 Diffusion Policy：用生成式分布建模多模态动作序列。
第15章 Flow Matching：用连续流场视角生成动作。
第16章 BC / ACT / Diffusion Policy / Flow Matching 对比：帮助读者形成模型选择判断。
```

因此，第三篇和第四篇的关系不是“经典方法讲完，再讲现代模型”，而是：

```text
第三篇说明为什么只看单步动作不够，
第四篇讨论在离线数据和真实机器人约束下，策略模型应该如何更强地表达动作分布和动作序列。
```
