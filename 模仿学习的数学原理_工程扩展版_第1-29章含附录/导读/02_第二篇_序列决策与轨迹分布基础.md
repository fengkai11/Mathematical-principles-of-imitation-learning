# 第二篇导读：序列决策与轨迹分布基础

第一篇已经把模仿学习最容易上手、也最容易误解的部分讲清楚了：Behavior Cloning 看起来像监督学习，分布偏移说明它为什么闭环会翻车，DAgger 则说明可以通过交互数据修正学习策略自己访问到的状态分布。

第一篇最后留下的真正问题是：

```text
BC 的训练分布和执行分布为什么会不一致？
策略为什么会把自己带到专家数据没有覆盖的地方？
单步动作像专家，为什么整条任务轨迹仍然可能失败？
```

第二篇的作用，就是把读者从“单步动作拟合”的视角，带到“序列决策与轨迹分布”的视角。

> **第二篇不是普通强化学习教材，而是为了解释模仿学习中的闭环问题：策略如何通过动作改变未来，如何诱导轨迹分布，以及为什么单步 loss 不能代表长期任务成功。**

读完这一篇后，读者应该不再只问：

```text
模型这一帧动作像不像专家？
```

而要开始追问：

```text
这个策略闭环执行后，会诱导出怎样的状态分布和轨迹分布？
它能否在自己造成的新状态里继续稳定行动？
单步动作误差和整条轨迹成功之间有什么差距？
同一状态下是否存在多种合理动作？
策略是否能表达这些不同动作模式？
隐变量和 CVAE 如何帮助我们训练这种多模态策略？
```

## 本篇在全书中的位置

第二篇承接第一篇，也为后续几篇打地基。

```text
第一篇：专家样本、BC、分布偏移、DAgger
        ↓
第二篇：MDP、轨迹分布、轨迹目标、概率策略、隐变量、CVAE
        ↓
第三篇：IRL、GAIL、Offline Imitation Learning
        ↓
第四篇：ACT、Diffusion Policy、Flow Matching
```

第一篇主要说明：为什么只在专家数据上做监督学习是不够的。

第二篇进一步说明：为什么模仿学习必须放回序列决策系统中理解。

第三篇会把这种理解推进到分布匹配与奖励视角：不只是动作像不像专家，而是策略访问到的状态—动作分布是否接近专家。

第四篇会进入现代机器人策略模型：ACT、Diffusion Policy、Flow Matching 等方法本质上都在更强地建模条件动作分布或动作序列分布。

## 第二篇与第一篇的术语桥接

第二篇不是从零开始，而是把第一篇中的概念进一步数学化。

| 第一篇术语 | 第二篇深化 | 读者应形成的理解 |
|---|---|---|
| 策略 $\pi_\theta$ | MDP 中的闭环决策规则 | 策略不是静态函数，放进环境后会改变未来 |
| 专家分布 $d^{\pi_E}$ | 专家策略诱导的访问分布 / 轨迹分布 | 专家数据来自专家策略走出来的轨迹 |
| 学习策略分布 $d^{\pi_\theta}$ | 学习策略诱导的访问分布 / 轨迹分布 | 模型部署后会形成自己的轨迹分布 |
| 单步 BC loss | 局部模仿信号 | 它适合训练初始策略，但不是最终任务目标 |
| 闭环失败 | 轨迹级目标未达成 | 失败发生在整条 rollout 上，不只是某一帧动作上 |
| 多种专家动作 | 多模态动作分布 | 同一状态下可能存在多条成功路径 |
| DAgger 数据聚合 | 修正学习策略访问分布上的数据覆盖 | 它把模型自己会遇到的状态补进训练集 |

## 本篇新增的数学对象

第二篇会集中引入并反复使用以下数学对象：

```text
MDP: M = (S, A, P, r, gamma)
状态: s_t
观测: o_t
动作: a_t
状态转移: P(s_{t+1}|s_t,a_t)
轨迹: tau = (s_0, a_0, s_1, a_1, ..., s_T)
策略: pi(a_t | s_t) 或 pi(a_t | o_t)
策略诱导的轨迹分布: p_pi(tau)
轨迹目标: J(pi)
确定性策略: a = f_theta(s)
概率策略: pi_theta(a|s)
隐变量: z
近似后验: q_phi(z | x, a)
条件生成分布: p_theta(a | x, z)
CVAE 的 ELBO
```

这些符号不是为了增加数学负担，而是为了解释几个工程上非常常见的现象：

```text
为什么 open-loop loss 很低，closed-loop 仍然失败？
为什么同一个状态下，专家动作可能有多个正确答案？
为什么 MSE 会把多种动作模式平均成一个坏动作？
为什么 ACT、Diffusion Policy 等方法要建模动作序列或动作分布？
为什么生成多个候选动作后仍然需要安全筛选？
```

## 本篇包含章节

### 第5章：MDP：模仿学习虽然不要 reward，但也逃不出序列决策

第5章补上序列决策地基。它要说明：模仿学习可以不显式设计 reward，但策略执行一定发生在 MDP 中。状态、动作、状态转移、策略和轨迹，是理解后续所有方法的共同语言。

这一章的核心判断是：

```text
模仿学习训练时看起来像监督学习，但执行时一定是序列决策。
```

### 第6章：从单步损失到轨迹损失：别只盯着眼前那一小步

第6章把评价尺度从单步动作误差推进到整条轨迹。它要说明：step-wise loss 可以训练初始策略，但不能完整代表任务成功。真正决定机器人能力的是闭环 rollout 后的轨迹结果。

这一章的核心判断是：

```text
单步 loss 是局部指标，轨迹目标才更接近机器人真实任务。
```

### 第7章：确定性策略与概率策略：机器人不要只会一个标准答案

第7章讨论策略输出到底应该是一个动作，还是一个动作分布。它要说明：在多模态动作场景中，同一个状态下可能有多个合理动作，确定性 MSE 容易学出平均动作。

这一章的核心判断是：

```text
如果数据里存在多个合理动作模式，策略就不能只被理解成一个确定函数。
```

### 第8章：隐变量策略：给机器人一个动作风格旋钮

第8章进一步引入隐变量 $z$。它要说明：多模态动作往往来自未被直接标注的风格、意图、阶段或策略模式。隐变量不是魔法，而是给这些隐藏因素一个数学容器。

这一章的核心判断是：

```text
隐变量 z 不是人类显式标签，而是模型用来解释动作模式差异的隐藏结构。
```

### 第9章：CVAE：训练时偷看答案，推理时假装自己懂了

第9章说明如何训练隐变量策略。训练数据通常只有条件和动作，没有 latent 标签。CVAE 用 encoder 在训练时根据 $(x,a)$ 推断 $z$，用 decoder 根据 $(x,z)$ 生成动作，并通过 ELBO 连接最大似然、重建损失和 KL 约束。

这一章的核心判断是：

```text
CVAE 的 encoder 是训练时的推断工具，decoder 才是推理时生成动作的策略主体。
```

## 本篇的数学主线

第二篇的数学主线可以概括为：

```text
第一篇得到：BC 在专家分布上最小化单步动作误差，闭环执行时会分布偏移
→ MDP: M=(S,A,P,r,gamma)
→ 状态转移: s_{t+1} ~ P(s_{t+1}|s_t,a_t)
→ 策略: a_t ~ pi(a_t|s_t) 或 pi(a_t|o_t)
→ 策略和转移共同诱导轨迹: tau ~ p_pi(tau)
→ 轨迹目标: J(pi)=E_{tau~p_pi}[C(tau)]
→ 单步 loss 与轨迹目标不一致
→ 确定性策略 a=f_theta(s) 的边界
→ 概率策略 pi_theta(a|s) 表达多种合理动作
→ 隐变量 z 表示动作模式
→ CVAE 用 encoder / decoder / ELBO 训练隐变量策略
```

更直白地说，本篇要完成一次视角升级：

```text
从“模仿学习是在专家数据上拟合动作”
升级为
“模仿学习是在序列决策系统中学习能诱导可靠轨迹分布的策略”。
```

## 统一工程主线：视觉引导机械臂抓取与放置

阅读本篇时，建议始终把公式落回一个统一工程任务：

```text
机器人通过相机观测桌面上的物体，输出末端位姿增量或动作序列，完成抓取并放置到目标区域。
```

在这个任务中：

- $s_t$ 可以理解为真实世界状态，例如物体位姿、机械臂状态、接触状态、夹爪是否打滑；
- $o_t$ 是机器人实际看到的观测，例如图像、本体状态、历史帧或视觉特征；
- $a_t$ 是策略输出的动作，例如末端速度、关节增量、夹爪开合命令；
- $P(s_{t+1} \mid s_t,a_t)$ 描述动作如何改变世界，例如物体被推走、夹爪打滑、接触状态变化；
- $p_\pi(\tau)$ 描述策略闭环执行后可能产生哪些轨迹；
- $J(\pi)$ 描述策略在自己诱导轨迹上的长期任务结果；
- $z$ 可以表示抓取风格、接近方向、动作节奏或任务阶段；
- CVAE 可以用来保留多个合理动作模式，而不是把它们平均掉。

## 读完本篇，你应该能判断什么？

读完第二篇后，你应该能做出以下工程判断：

1. **判断一个问题是否需要 MDP 视角**：只要当前动作会影响下一刻观测和状态，就不能只按独立样本预测理解。
2. **判断 open-loop loss 的边界**：验证集动作误差低，只能说明模型在专家分布上会答题，不说明闭环一定可靠。
3. **判断何时需要轨迹级评价**：抓取、插孔、泊车、绕障等任务，都必须看 rollout 成功率、失败类型和安全指标。
4. **判断确定性策略是否足够**：如果一个状态下基本只有一个正确动作，确定性策略可能够用；如果存在多种合理动作，概率策略或生成式策略更合适。
5. **判断 MSE 平均动作风险**：当数据中存在左抓/右抓、左绕/右绕、一把入库/多把修正等模式时，平均动作可能是坏动作。
6. **判断隐变量是否有必要**：如果动作差异来自隐藏风格、任务阶段或未标注意图，latent 可能比单一动作头更自然。
7. **判断 CVAE 训练和推理是否一致**：训练时 encoder 可以看专家动作，推理时不能；如果评测时偷看真实动作，就是信息泄漏。
8. **判断生成式策略是否还需要安全模块**：CVAE 生成多样候选动作，不代表候选动作都安全，真实部署仍需要碰撞、限位、平滑性和 fallback 检查。
