# 第一篇导读：模仿学习的基本问题

这一篇回答“模仿学习到底是什么，以及为什么最朴素的行为克隆会在闭环中失败”。读完这一篇，读者应该能够区分开环拟合与闭环执行，并理解 DAgger 为什么不是普通数据增强。

更重要的是，第一篇要建立两条互相配合的主线。

第一条是读者的**认知主线**，它从普通工程师熟悉的监督学习出发：

```text
普通监督学习
→ i.i.d. 假设：训练分布和测试分布近似一致
→ Behavior Cloning 借用监督学习形式
→ 但 BC 输出动作，动作会改变未来输入
→ i.i.d. 或近似同分布前提失效
→ 专家分布 d^{pi_E} 与学习策略分布 d^{pi_theta} 不一致
→ 单步误差在闭环中累积
→ DAgger 用学习策略访问到的状态修正训练分布
```

第二条是贯穿后续公式的**数学主线**：

```text
专家轨迹数据 D_E
→ 学习策略 pi_theta(a | o)
→ BC 在专家分布 d^{pi_E}(o) 上最小化单步损失
→ 部署时策略面对自己诱导出的分布 d^{pi_theta}(o)
→ 分布不一致导致误差累积
→ DAgger 用 d^{pi_i}(o) 上的新标注修正训练分布
```

因此，阅读第一篇时不要只记住“BC 是监督学习”“DAgger 是交互采样”。更应该追问：

```text
普通监督学习为什么通常可以依赖固定训练集？
BC 为什么表面像监督学习，但闭环执行时又不像普通监督学习？
这个方法在什么分布上训练？
部署时策略真正面对什么分布？
单步 loss 和长时闭环成功率之间差了什么？
DAgger 到底改变了 loss，还是改变了数据来源？
```

## 本篇包含章节

- 第1章：模仿学习到底在模仿什么
- 第2章：Behavior Cloning：最朴素也最容易翻车的模仿学习
- 第3章：分布偏移：为什么 Behavior Cloning 会越走越歪
- 第4章：DAgger：让老师坐副驾，专治模型越走越歪

在正式进入第 1 章之前，建议先阅读新增的“第一篇数学主线：从专家轨迹到执行分布”。它把四章中的核心公式提前串成一条线，帮助读者在阅读具体章节时知道每个公式放在全局中的位置。

## 本篇在全书中的作用

建立“普通监督学习 → i.i.d. 假设 → 行为克隆 → 分布偏移 → DAgger”的入门主线，让读者看到模仿学习为什么不能只按普通监督学习理解。

从全书结构看，本篇不是为了把 MDP、轨迹分布、隐变量策略、生成式动作模型一次讲完，而是为了先回答一个更基础的问题：

> **为什么一个在专家数据上训练得很好的策略，上机闭环执行时仍然可能失败？**

这个问题会在后续反复出现：

- 第 5–6 章会把它推进到 MDP 和轨迹损失；
- 第 7–9 章会把它推进到概率策略、隐变量和 CVAE；
- 第 10–12 章会把它推进到分布匹配、IRL/GAIL 和离线模仿学习；
- 第 13–16 章会把它推进到 ACT、Diffusion Policy、Flow Matching 与方法选型；
- 第 25–29 章会把它推进到实机部署、OPE、C4/ADR 和数据闭环。

所以，第一篇的数学重点不是公式数量，而是公式之间的因果关系：

```text
D_E 是训练材料；
pi_theta 是学习目标；
d^{pi_E} 是专家数据来自的分布；
d^{pi_theta} 是模型部署时真正面对的分布；
DAgger 的 D_i 是把训练数据逐步推向学习策略访问分布的工具。
```

## 阅读提醒

阅读本篇时，不要只关注模型名字，而要始终追问三个问题：本篇新增了什么数学对象？它解决了前一章留下的什么问题？它又为后一篇铺垫了什么能力？

尤其要记住一句话：

> **模仿学习的核心困难，不只是动作预测误差，而是普通监督学习的训练/测试近似同分布前提，在闭环动作改变未来输入后失效了。这个失效最终表现为训练分布、执行分布和闭环时间累积之间的错配。**
