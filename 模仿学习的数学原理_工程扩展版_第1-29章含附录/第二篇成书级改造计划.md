## 一、第二篇总体定位

第一篇已经建立了核心问题：

```text
BC 表面是监督学习，
但策略动作会改变未来输入，
所以训练分布和执行分布不一致，
导致误差累积；
DAgger 通过收集学习策略自己会遇到的状态，
把这些状态重新纳入训练数据。
```

第二篇不能写成普通强化学习教材，而要服务于模仿学习主线，回答第一篇留下的更底层问题：

```text
为什么策略动作会改变未来输入？
这个“改变未来”的过程如何被数学描述？
策略执行后为什么会诱导轨迹分布？
为什么单步动作 loss 不等于长期任务成功？
为什么同一状态下可能有多个合理动作？
为什么需要概率策略、隐变量和 CVAE？
```

因此，第二篇的定位是：

> **从“单步动作拟合”升级到“序列决策、轨迹分布和多模态动作建模”的数学基础。**

## 三、第二篇数学对象主线

第二篇必须显式推进以下数学对象：

| 数学对象 | 符号 | 一句话含义 | 工程对应物 | 后续作用 |
|---|---|---|---|---|
| 状态 | $s_t$ | 环境和机器人在时刻 $t$ 的真实情况 | 物体位姿、机械臂状态、接触状态、是否抓稳 | 建立 MDP 与状态转移 |
| 观测 | $o_t$ | 策略实际能看到的信息 | 图像、本体状态、历史帧、视觉特征 | 连接真实系统和策略输入 |
| 动作 | $a_t$ | 策略输出的控制命令 | 末端增量、关节命令、夹爪命令 | 改变下一时刻状态 |
| 转移模型 | $P(s_{t+1}\mid s_t,a_t)$ | 动作如何改变未来状态 | 机器人动力学、接触、打滑、物体移动 | 解释闭环分布来源 |
| MDP | $M=(S,A,P,r,\gamma)$ | 序列决策的数学框架 | 机器人在环境中连续执行任务 | 第5章核心对象 |
| 轨迹 | $\tau$ | 一次闭环执行的状态—动作序列 | 一次抓取、放置、泊车或绕障 rollout | 第6章核心对象 |
| 轨迹分布 | $p_\pi(\tau)$ | 策略在环境中可能产生哪些轨迹 | 不同执行结果、成功/失败回放集合 | 解释长期目标 |
| 轨迹目标 | $J(\pi)$ | 策略在整条轨迹上的目标 | 成功率、安全、平滑、任务完成 | 连接单步 loss 与长期成功 |
| 概率策略 | $\pi(a\mid s)$ 或 $\pi(a\mid o)$ | 给定状态或观测下的动作分布 | 多个候选动作、不同抓取方式 | 第7章核心对象 |
| 隐变量 | $z$ | 未显式标注的风格、意图或动作模式 | 左抓/右抓、快/慢、接近方向 | 第8章核心对象 |
| 近似后验 | $q_\phi(z\mid x,a)$ | 训练时从条件和动作推断隐变量 | 训练时“偷看答案”提取动作模式 | 第9章 CVAE encoder |
| 条件生成分布 | $p_\theta(a\mid x,z)$ | 给定条件和隐变量生成动作 | decoder 产生动作候选 | 第9章 CVAE decoder |
| ELBO | ELBO | 可训练的变分下界目标 | 重建动作 + 约束隐变量分布 | 第9章核心推导 |

## 四、第二篇公式主线

第二篇公式主线建议统一为：

```text
第一篇得到：BC 在专家分布上最小化单步动作误差，闭环执行时会分布偏移
→ 第5章引入 MDP：M=(S,A,P,r,gamma)
→ 状态转移：s_{t+1} ~ P(s_{t+1}|s_t,a_t)
→ 策略：a_t ~ pi(a_t|s_t) 或 pi(a_t|o_t)
→ 策略和转移共同诱导轨迹：tau ~ p_pi(tau)
→ 第6章引入轨迹目标：J(pi)=E_{tau~p_pi}[C(tau)] 或 E[R(tau)]
→ 单步 loss 与轨迹目标不一致
→ 第7章说明确定性策略 a=f_theta(s) 的边界
→ 概率策略 pi_theta(a|s) 表达多种合理动作
→ MSE 在多模态数据下容易学条件均值
→ 第8章引入隐变量 z 表示动作模式
→ 第9章用 CVAE 训练隐变量策略：encoder q_phi(z|x,a)，decoder p_theta(a|x,z)
→ ELBO 连接重建损失、KL 约束和条件动作生成
```

注意：第二篇的公式主线必须围绕“模仿学习为什么需要序列决策与动作分布建模”展开，不要变成普通 RL 公式罗列。

## 五、各章节改造计划

## 5.1 第5章：MDP：模仿学习虽然不要 reward，但也逃不出序列决策

### 本章角色

第5章要承接第一篇的“动作会改变未来输入”，给出描述这个闭环因果关系的数学框架。

本章不是普通 RL 入门，而是回答：

```text
为什么模仿学习即使不显式设计 reward，也必须理解 MDP？
为什么策略执行不是静态预测，而是发生在状态转移系统中？
```

### 必须强化的主线

```text
第一篇：BC 输出动作，动作改变下一时刻观测
→ 第5章：用 MDP 描述“动作如何改变未来”
→ 状态、动作、转移、策略共同决定闭环执行过程
→ 第6章自然进入轨迹分布和轨迹损失
```

### 核心数学对象

```text
s_t, o_t, a_t, S, A, P, r, gamma, pi, tau
```

### 核心公式

1. MDP 定义：

```text
M = (S, A, P, r, gamma)
```

2. 状态转移：

```text
s_{t+1} ~ P(s_{t+1}|s_t,a_t)
```

3. 策略：

```text
a_t ~ pi(a_t|s_t)
```

或在部分可观测系统中：

```text
a_t ~ pi(a_t|o_t)
```

4. 一条轨迹：

```text
tau = (s_0,a_0,s_1,a_1,...,s_T)
```

### 必须补的证明 / 推导

建议写成命题：

> **命题：策略和状态转移共同决定下一时刻状态分布。**

证明思路：策略先根据当前状态或观测产生动作，转移模型再根据状态和动作产生下一状态。因此，未来输入不是外部固定给定，而是被策略动作间接决定。

### 工程解释

机械臂抓取中：

```text
s_t：物体真实位姿、夹爪真实位置、接触状态；
o_t：相机图像、本体状态；
a_t：末端位姿增量或夹爪命令；
P：动作执行后，物体是否被推走、是否打滑、夹爪是否夹稳。
```

### 常见误解

| 常见误解 | 正确认识 |
|---|---|
| 模仿学习不用 reward，所以不用 MDP | 即使不设计 reward，策略执行仍发生在 MDP 中 |
| MDP 是 RL 专属概念 | MDP 是描述闭环序列决策的通用框架 |
| 状态和观测是一回事 | 状态是世界真实情况，观测是策略实际看到的信息 |
| 转移模型必须被显式学习 | 不一定显式学习，但它客观决定动作如何改变未来 |

### 章节接力句

第5章结尾要自然引出第6章：

```text
既然策略和环境转移会共同产生一整条轨迹，那么评价策略就不能只看某一步动作像不像专家，还要看整条轨迹是否成功。这就是第6章要讨论的单步损失与轨迹损失问题。
```

## 5.2 第6章：从单步损失到轨迹损失：别只盯着眼前那一小步

### 本章角色

第6章要把视角从“单步动作误差”推进到“整条轨迹结果”。

本章回答：

```text
为什么 open-loop 单步 loss 低，不代表 rollout 成功？
为什么真实机器人任务更关心轨迹级结果？
```

### 必须强化的主线

```text
第5章：MDP 中策略和转移诱导轨迹
→ 第6章：策略目标应该在轨迹上定义
→ 单步 loss 是局部训练信号
→ 轨迹目标更接近真实任务成功
→ 第7章进一步讨论单步动作本身也可能不是唯一答案
```

### 核心数学对象

```text
tau, p_pi(tau), J(pi), C(tau), R(tau), step-wise loss, trajectory-level loss
```

### 核心公式

1. 策略诱导轨迹分布：

```text
p_pi(tau)
```

2. 轨迹目标：

```text
J(pi) = E_{tau ~ p_pi}[R(tau)]
```

或以代价形式写：

```text
J(pi) = E_{tau ~ p_pi}[C(tau)]
```

3. 单步 BC loss：

```text
E_{(o,a_E) ~ D_E}[ell(pi(o),a_E)]
```

4. 目标错配：

```text
单步动作误差小，不等于轨迹代价小
```

### 必须补的证明 / 推导

建议写成命题：

> **命题：单步动作损失不等价于轨迹级任务目标。**

证明思路：单步 loss 只评价专家分布附近某个观测下动作是否接近；轨迹目标评价策略闭环执行后整条轨迹的结果。由于动作会影响后续状态，两者评价对象不同。

可以用反例说明：

```text
每一步动作只偏一点，但连续偏差导致最后抓空；
某一步动作与专家不同，但仍然通过另一条合理路径完成任务。
```

### 工程解释

机械臂抓取中：

```text
open-loop loss：每帧动作是否接近专家；
closed-loop success：是否真的抓起并放入目标位置；
trajectory cost：碰撞、打滑、失败、用时、动作平滑性等综合结果。
```

### 常见误解

| 常见误解 | 正确认识 |
|---|---|
| 单步 loss 低就代表策略好 | 只能说明专家数据附近动作像，不保证闭环成功 |
| 轨迹目标只和强化学习有关 | 任何闭环机器人任务最终都要看轨迹结果 |
| 轨迹级评价可以完全替代训练 loss | 训练仍需要可优化信号，评价和训练目标可以不同 |
| 和专家动作不同就是错 | 在多解任务中，另一条动作轨迹也可能成功 |

### 章节接力句

第6章结尾要自然引出第7章：

```text
第6章说明了单步 loss 不等于轨迹成功。但还有一个更细的问题：即使只看当前状态，专家也未必只有一个正确动作。如果同一状态下有多种合理动作，确定性策略和 MSE 会遇到新的问题。这就是第7章的主题。
```

## 5.3 第7章：确定性策略与概率策略：机器人不要只会一个标准答案

### 本章角色

第7章要从“轨迹目标”进一步推进到“动作分布”。它要说明：同一状态下可能有多个合理动作，策略不一定应该输出唯一标准答案。

本章回答：

```text
为什么确定性策略在多模态动作数据上会失败？
为什么 MSE 容易学到平均动作？
为什么需要概率策略？
```

### 必须强化的主线

```text
第6章：轨迹任务可能有多条成功路径
→ 第7章：同一状态下可能有多个合理动作
→ 确定性策略 a=f_theta(s) 只能输出一个动作
→ MSE 倾向于条件均值
→ 概率策略 pi(a|s) 能表达动作分布
→ 第8章进一步用隐变量解释动作模式差异
```

### 核心数学对象

```text
f_theta(s), pi_theta(a|s), conditional mean, multi-modal action distribution
```

### 核心公式

1. 确定性策略：

```text
a = f_theta(s)
```

2. 概率策略：

```text
a ~ pi_theta(a|s)
```

3. MSE 目标：

```text
min_f E[||A - f(S)||^2]
```

4. 条件均值结论：

```text
f^*(s) = E[A|S=s]
```

### 必须补的证明 / 推导

必须写成命题：

> **命题：MSE 下确定性策略的最优预测是条件均值。**

这是第二篇最关键的证明之一。

证明之后必须解释：

```text
如果同一状态下左抓和右抓都合理，MSE 可能学习到二者中间的平均动作；
这个平均动作未必是任何一个专家真正会执行的动作；
因此确定性 MSE 在多模态动作场景中存在结构性风险。
```

### 工程解释

机械臂抓取中：

```text
杯子可以从左侧抓，也可以从右侧抓；
轴承套可以先向左绕开，也可以先向右绕开；
MSE 平均后可能让夹爪从中间撞过去。
```

### 常见误解

| 常见误解 | 正确认识 |
|---|---|
| MSE 小就是动作合理 | 多模态下均值动作可能不可执行 |
| 概率策略只是为了加随机性 | 概率策略是为了表达多个合理动作模式 |
| 确定性策略一定不好 | 单峰、短 horizon、低歧义任务中仍然有效 |
| 多模态问题只要加数据就能解决 | 如果训练目标仍强制输出一个均值动作，问题可能仍存在 |

### 章节接力句

第7章结尾要自然引出第8章：

```text
概率策略说明动作不一定只有一个标准答案。但如果我们希望模型不仅知道“动作有多个模式”，还要能区分这些模式背后的风格、意图或阶段，就需要引入一个新的数学容器：隐变量 z。
```

## 5.4 第8章：隐变量策略：给机器人一个动作风格旋钮

### 本章角色

第8章要把概率策略中的“多种合理动作”进一步结构化，用隐变量表示未被显式标注的动作模式。

本章回答：

```text
多模态动作的差异来自哪里？
如果数据里没有显式标注“左抓/右抓”，模型如何表达这种隐藏模式？
隐变量 z 到底是什么，不是什么？
```

### 必须强化的主线

```text
第7章：概率策略可以表达动作分布
→ 第8章：动作分布中的模式差异可以由隐变量 z 表示
→ z 可以代表风格、意图、阶段或路径选择
→ 第9章需要 CVAE 来训练这种带隐变量的策略
```

### 核心数学对象

```text
z, p(z), p_theta(a|x,z), p_theta(a|x), latent mode, marginalization
```

其中 $x$ 可以统一表示策略条件输入，例如观测、状态或历史：

```text
x = o, s, 或历史观测 h
```

### 核心公式

1. 隐变量先验：

```text
z ~ p(z)
```

2. 条件动作生成：

```text
a ~ p_theta(a|x,z)
```

3. 边缘化动作分布：

```text
p_theta(a|x) = int p_theta(a|x,z)p(z) dz
```

### 必须补的证明 / 推导

建议写成命题：

> **命题：隐变量策略通过边缘化表达多模态动作分布。**

推导重点：不同 $z$ 对应不同动作模式，对 $z$ 积分或求和后，整体 $p(a|x)$ 可以形成多峰分布。

### 工程解释

机械臂抓取中：

```text
z 可以对应左侧接近 / 右侧接近；
z 可以对应快抓 / 慢抓；
z 可以对应先对准 / 先绕障；
z 可以对应不同任务阶段或不同风格。
```

### 常见误解

| 常见误解 | 正确认识 |
|---|---|
| z 是人工标注标签 | z 通常不是显式标签，而是模型学习出的隐藏结构 |
| z 一定可解释 | 有时可解释，有时只是有用的潜在表示 |
| 加隐变量就一定解决多模态 | 还需要合适训练目标和推理采样机制 |
| z 越大越好 | 维度过大可能导致训练不稳定或 posterior collapse |

### 章节接力句

第8章结尾要自然引出第9章：

```text
隐变量策略给了我们表达多种动作模式的数学形式，但训练数据通常没有 z 的标签。下一章要回答：在没有 latent 标签的情况下，如何训练一个能使用隐变量生成动作的策略？这就是 CVAE。
```

## 5.5 第9章：CVAE：训练时偷看答案，推理时假装自己懂了

### 本章角色

第9章要给出隐变量策略的可训练方案。它承接第8章的隐变量建模问题，引入 encoder、decoder 和 ELBO。

本章回答：

```text
没有 z 标签，如何训练隐变量策略？
训练时 encoder 为什么可以看动作？
推理时为什么不能看动作？
ELBO 里的重建项和 KL 项分别约束什么？
```

### 必须强化的主线

```text
第8章：隐变量 z 可以表达动作模式
→ 但训练数据没有 z 标签
→ CVAE 训练时用 q_phi(z|x,a) 从动作反推 z
→ decoder 用 p_theta(a|x,z) 学会根据条件和 z 生成动作
→ ELBO 让模型既能重建动作，又让 posterior 接近 prior
→ 推理时只采样 z ~ p(z)，再用 decoder 生成动作
```

### 核心数学对象

```text
x, a, z, p(z), q_phi(z|x,a), p_theta(a|x,z), p_theta(a|x), ELBO, KL
```

### 核心公式

1. 条件边缘似然：

```text
p_theta(a|x) = int p_theta(a|x,z)p(z) dz
```

2. 近似后验：

```text
q_phi(z|x,a)
```

3. ELBO：

```text
log p_theta(a|x) >= E_{z~q_phi(z|x,a)}[log p_theta(a|x,z)] - KL(q_phi(z|x,a) || p(z))
```

4. 训练目标：

```text
重建项 + KL 项
```

### 必须补的证明 / 推导

必须给出 ELBO 的基础推导。

推导要求：

```text
从 log p_theta(a|x) 开始；
引入 q_phi(z|x,a)；
写成期望形式；
使用 Jensen 不等式得到下界；
解释重建项和 KL 项；
说明训练时 encoder 可看动作，推理时不能看动作。
```

如果正文篇幅过长，可正文保留核心推导，附录 G 或附录 B 承接 Jensen / KL 基础。

### 工程解释

机械臂抓取中：

```text
x：当前图像、本体状态、任务条件；
a：专家动作或动作片段；
z：隐藏抓取模式；
encoder：训练时根据“当前条件 + 专家动作”推断这次演示属于哪种模式；
decoder：根据“当前条件 + 模式 z”生成动作；
推理时：没有专家动作，只能从 prior 采样 z，再生成候选动作。
```

### 常见误解

| 常见误解 | 正确认识 |
|---|---|
| encoder 是部署时策略主体 | decoder 才是推理时生成动作的主体 |
| 推理时还能把真实动作喂给 encoder | 这是信息泄漏，真实部署没有专家动作 |
| KL 项只是正则化 | KL 约束 posterior 接近 prior，使推理时采样可用 |
| 重建 loss 低就代表闭环好 | 仍需闭环 rollout、安全筛选和分布覆盖 |
| CVAE 生成多个动作就都可执行 | 候选动作仍需碰撞、限位、平滑和任务约束检查 |
