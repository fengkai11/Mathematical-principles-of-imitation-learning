# 全书统一术语表

> 本术语表用于统一全书中的常用概念、英文名称、中文译法和简要说明。阅读时遇到某个术语拿不准含义，可以先回到本表快速定位。

## 1. 基础对象与数据

| 中文术语 | 英文术语 | 建议记号 | 统一说明 |
|---|---|---|---|
| 状态 | State | <span class="math">\\(s\_t\\)</span> | 环境在时刻 <span class="math">\\(t\\)</span> 的真实内部状态。 |
| 观测 | Observation | <span class="math">\\(o\_t\\)</span> | 智能体在时刻 <span class="math">\\(t\\)</span> 实际能看到的输入，如图像、激光、关节状态。 |
| 动作 | Action | <span class="math">\\(a\_t\\)</span> | 策略输出的控制命令。 |
| 轨迹 | Trajectory | <span class="math">\\(\tau\\)</span> | 一次完整交互过程，通常写作 <span class="math">\\((s\_0,a\_0,s\_1,a\_1,\dots,s\_T)\\)</span>。 |
| 专家示范 | Expert Demonstration | <span class="math">\\(\mathcal{D}\_E\\)</span> | 由专家行为构成的数据集。 |
| 专家策略 | Expert Policy | <span class="math">\\(\pi\_E\\)</span> | 生成专家动作的策略。 |
| 数据集 | Dataset | <span class="math">\\(\mathcal{D}\\)</span> | 用于训练或评估的样本集合。 |
| 动作块 | Action Chunk | <span class="math">\\(a\_{t:t+H-1}\\)</span> | 一次性预测的多步动作序列。 |
| 历史窗口 | History Window | — | 模型用于决策的历史观测和动作片段。 |

## 2. 策略与分布

| 中文术语 | 英文术语 | 建议记号 | 统一说明 |
|---|---|---|---|
| 策略 | Policy | <span class="math">\\(\pi\_\theta\\)</span> | 从状态或观测到动作分布的映射。 |
| 确定性策略 | Deterministic Policy | <span class="math">\\(a=f\_\theta(o)\\)</span> | 每个输入只输出一个动作。 |
| 概率策略 | Stochastic Policy | <span class="math">\\(\pi\_\theta(a\mid o)\\)</span> | 给定输入后输出动作分布。 |
| 条件分布 | Conditional Distribution | <span class="math">\\(p(a\mid o)\\)</span> | 在给定条件下某个随机变量的分布。 |
| 轨迹分布 | Trajectory Distribution | <span class="math">\\(p\_\pi(\tau)\\)</span> | 在策略 <span class="math">\\(\pi\\)</span> 下生成整条轨迹的概率分布。 |
| 状态访问分布 | State Visitation Distribution | <span class="math">\\(d\_\pi(s)\\)</span> | 策略在执行中访问到各状态的频率分布。 |
| 占用测度 | Occupancy Measure | <span class="math">\\(\rho\_\pi(s,a)\\)</span> | 状态—动作对在轨迹中的访问频率度量。 |
| 多模态动作分布 | Multimodal Action Distribution | — | 一个状态下可能存在多种都合理的动作模式。 |
| 分布偏移 | Distribution Shift | — | 训练分布和执行分布不一致。 |
| 协变量偏移 | Covariate Shift | — | 由于策略改变导致输入状态分布发生变化。 |

## 3. 损失函数与统计量

| 中文术语 | 英文术语 | 建议记号 | 统一说明 |
|---|---|---|---|
| 损失函数 | Loss Function | <span class="math">\\(L(\theta)\\)</span> | 衡量模型预测与目标差异的函数。 |
| 最大似然估计 | Maximum Likelihood Estimation | MLE | 通过最大化观测数据出现概率来估计参数。 |
| 负对数似然 | Negative Log-Likelihood | NLL | 最大似然常用等价损失形式。 |
| 交叉熵 | Cross Entropy | — | 离散动作常用监督损失。 |
| 均方误差 | Mean Squared Error | MSE | 连续动作回归的常见损失。 |
| KL 散度 | Kullback–Leibler Divergence | <span class="math">\\(D\_{KL}(p\|q)\\)</span> | 衡量两个分布差异的常用量。 |
| 熵 | Entropy | <span class="math">\\(H(p)\\)</span> | 衡量分布不确定性的指标。 |
| 最大熵原理 | Maximum Entropy Principle | — | 在约束下选择最不额外做假设的分布。 |
| ELBO | Evidence Lower Bound | ELBO | VAE/CVAE 中用于训练的下界目标。 |
| Score Function | Score Function | <span class="math">\\(\nabla\_x \log p(x)\\)</span> | 分布对输入的对数密度梯度。 |
| Score Matching | Score Matching | — | 通过匹配 score 来学习分布的方法。 |

## 4. 经典模仿学习方法

| 中文术语 | 英文术语 | 统一说明 |
|---|---|---|
| 行为克隆 | Behavior Cloning, BC | 把模仿学习视为监督学习，直接拟合专家动作。 |
| 数据聚合 | Dataset Aggregation, DAgger | 让当前策略自己跑，再让专家补标，从而修复分布偏移。 |
| 逆强化学习 | Inverse Reinforcement Learning, IRL | 从专家行为中反推潜在奖励函数。 |
| 最大熵逆强化学习 | Maximum Entropy IRL | 假设专家轨迹服从最大熵形式的分布。 |
| 生成式对抗模仿学习 | Generative Adversarial Imitation Learning, GAIL | 通过对抗训练直接匹配专家与策略行为分布。 |
| 离线模仿学习 | Offline Imitation Learning | 只用固定离线数据训练策略，不再与环境在线交互。 |

## 5. 现代策略建模方法

| 中文术语 | 英文术语 | 统一说明 |
|---|---|---|
| 条件变分自编码器 | Conditional Variational Autoencoder, CVAE | 用隐变量建模动作多样性与风格。 |
| 动作块变换器 | Action Chunking with Transformers, ACT | 用 Transformer 一次预测一段动作。 |
| 扩散策略 | Diffusion Policy | 用扩散生成过程建模复杂动作分布。 |
| 决策 Transformer | Decision Transformer | 把决策建模写成序列建模问题。 |
| Transformer 策略 | Transformer Policy | 用自注意力统一建模历史上下文和动作预测。 |
| 视觉—语言—动作模型 | Vision-Language-Action, VLA | 将视觉、语言和动作统一到同一大模型框架中。 |
| 世界模型 | World Model | 显式或隐式学习环境动态与可预测结构。 |

## 6. 机器人与应用语境

| 中文术语 | 英文术语 | 统一说明 |
|---|---|---|
| 开环评测 | Open-loop Evaluation | 不让策略真正闭环控制，只离线比较预测结果。 |
| 闭环执行 | Closed-loop Execution | 策略输出会影响后续状态，需要真实滚动执行。 |
| 失效恢复 | Recovery | 系统偏离理想状态后重新回到可控轨道的能力。 |
| 安全约束 | Safety Constraint | 在实机部署中对动作、状态和系统行为施加的限制。 |
| 回退机制 | Fallback | 当策略不可靠时切回传统控制器或安全模式。 |
| 数据覆盖度 | Coverage | 训练数据对真实任务空间的覆盖范围。 |
| 分布外输入/动作 | Out-of-Distribution, OOD | 超出训练数据支持范围的输入或动作。 |

## 7. 使用建议

1. 如果某一章的数学推导看起来复杂，优先回到本术语表确认对象、分布和损失分别在说什么。
2. 阅读现代方法（ACT、Diffusion Policy、VLA）时，先抓住“它是在建模什么分布”，再看网络结构。
3. 术语表的目的不是替代正文，而是帮助全书在概念层面保持统一语言。

|---|---|---|---|
| 流匹配 | Flow Matching | <span class="math">\\(\mathcal{L}\_{FM}\\)</span> | 通过回归连续速度场，把简单噪声分布搬运到数据分布的生成建模方法。 |
| 速度场 | Vector Field / Velocity Field | <span class="math">\\(v\_\theta(x,t,c)\\)</span> | 指示样本在生成时间 <span class="math">\\(t\\)</span> 应该如何移动的向量函数。 |
| 连续归一化流 | Continuous Normalizing Flow, CNF | — | 用连续动力系统描述分布变换的生成模型。 |
| 世界模型 | World Model | <span class="math">\\(\hat z\_{t+1}=F(z\_t,a\_t)\\)</span> | 预测动作如何改变未来状态或未来表示的模型。 |
| 世界动作模型 | World Action Model, WAM | — | 同时建模未来世界表示和机器人动作的具身模型。 |

| 前向动力学 | Forward Dynamics | <span class="math">\\(F(z\_t,a\_t)\\)</span> | 根据当前状态和动作预测未来状态。 |
| 逆动力学 | Inverse Dynamics | <span class="math">\\(I(z\_t,z\_{t+1})\\)</span> | 根据状态变化反推可能动作。 |
| 状态空间模型 | State Space Model, SSM | <span class="math">\\(h\_k=Ah\_{k-1}+Bx\_k\\)</span> | 用隐藏状态递推建模长序列的模型族。 |
| Mamba | Mamba | — | 基于选择性状态空间模型的线性时间序列架构。 |
| 选择性状态更新 | Selective State Update | <span class="math">\\(A(x),B(x),C(x)\\)</span> | 让状态空间参数依赖当前输入，从而选择性保留或遗忘信息。 |
| 偏好对齐 | Preference Alignment | — | 利用人类偏好数据调整策略行为边界。 |

| 参考策略 | Reference Policy | <span class="math">\\(\pi\_{ref}\\)</span> | DPO 中用于约束新策略不要过度偏离的基准策略。 |

## 8. 新版布局新增术语

| 中文术语 | 英文术语 | 建议记号 | 统一说明 |
|---|---|---|---|
| 快慢模型 | Fast-Slow Model / Dual-System Architecture | — | 将机器人策略拆成低频慢模型与高频快模型；慢模型负责理解、规划、预测，快模型负责低延迟连续动作控制。 |
| 慢模型 | Slow Model / System 2 | <span class="math">\\(g\_\phi\\)</span> | 通常处理语言、视觉语义、世界预测、子目标规划或价值判断。 |
| 快模型 | Fast Model / System 1 | <span class="math">\\(\pi\_\theta\\)</span> | 通常负责 action chunk、连续控制、动作生成和实时闭环。 |
| 高层条件 | High-level Condition | <span class="math">\\(z\_k\\)</span> | 慢模型传给快模型的子目标、计划、latent intention 或 goal representation。 |
| 异步条件缓存 | Asynchronous Condition Buffer | <span class="math">\\(\bar z\_t\\)</span> | 快模型在高频执行时使用最近一次可用的慢模型输出。 |
| 控制周期 | Control Period | <span class="math">\\(\Delta t\_{\text{fast}}\\)</span> | 快模型每次输出控制命令所对应的时间间隔。 |
| 后训练 | Post-training | — | 在预训练或基础训练之后，用偏好、纠偏、接管、失败恢复等数据进一步优化模型。 |
| 人类接管纠偏 | Human Intervention / Takeover Correction | — | 人类在机器人执行失败或有风险时介入，形成偏好数据或修正轨迹。 |

## 9. 第八篇新增工程术语

| 中文术语 | 英文术语 | 建议记号 | 统一说明 |
|---|---|---|---|
| 仿真到现实 | Sim-to-Real | — | 从仿真环境训练或验证的策略迁移到真实硬件执行。 |
| 域随机化 | Domain Randomization | <span class="math">\\(\xi \sim p(\xi)\\)</span> | 在训练时随机化动力学、视觉、噪声、摩擦等参数，提高真实环境鲁棒性。 |
| 安全集 | Safe Set | <span class="math">\\(\mathcal{S}\_{safe}\\)</span> | 系统允许进入的安全状态集合。 |
| 安全投影 | Safety Projection | <span class="math">\\(a^{safe}\\)</span> | 将模型原始动作投影到安全可行域。 |
| 离线策略评估 | Off-Policy Evaluation, OPE | <span class="math">\\(V^\pi\\)</span> | 用行为策略收集的数据评估目标策略的价值。 |
| 行为策略 | Behavior Policy | <span class="math">\\(\mu\\)</span> | 采集日志数据时使用的策略。 |
| 目标策略 | Target Policy | <span class="math">\\(\pi\\)</span> | 当前希望评估或部署的新策略。 |
| 重要性采样 | Importance Sampling | <span class="math">\\(\rho\\)</span> | 通过密度比修正行为策略与目标策略的数据分布差异。 |
| 双重鲁棒估计 | Doubly Robust Estimation | DR | 结合模型估计和重要性采样校正，降低 OPE 的偏差或方差风险。 |
| C4 模型 | C4 Model | — | 用 Context、Container、Component、Code 多层视图描述软件系统架构。 |
| 架构决策记录 | Architecture Decision Record, ADR | — | 记录关键工程架构选择、备选方案和后果的轻量文档。 |
| 影子模式 | Shadow Mode | — | 新策略只输出建议动作，不真实控制机器人，用于上线前验证。 |
| 灰度部署 | Gradual Rollout | <span class="math">\\(\lambda\\)</span> | 按比例逐步扩大新策略控制范围，降低上线风险。 |
| 回滚 | Rollback | — | 当风险指标超阈值时切回旧模型或安全策略。 |
| 数据闭环 | Data Flywheel / Data Loop | — | 采集、训练、评估、部署、监控、接管、再训练的持续迭代系统。 |
