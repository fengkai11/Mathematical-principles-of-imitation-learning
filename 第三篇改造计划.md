> 本文档按照《成书级章节改造方法.md》整理，用于指导《模仿学习的数学原理》第三篇“经典模仿学习的分布匹配与奖励视角”的成书级改造。

第三篇当前结构：

## 一、第三篇总目标

第三篇不能写成：

```text
第10章：介绍 IRL
第11章：介绍 GAIL
第12章：介绍 Offline IL
```

而应该收敛成一条数学主线：

```text
第二篇已经说明：
策略会诱导轨迹分布，单步 imitation loss 不等于轨迹级成功。

第三篇继续追问：
如果单步动作拟合不够，我们能不能从更高层次理解专家行为？

于是引出：
轨迹回报 R(tau)
→ 奖励函数 r(s,a)
→ 专家偏好与 reward ambiguity
→ occupancy measure rho_pi(s,a)
→ 分布匹配
→ 离线数据覆盖、support mismatch、OOD action

最后为第四篇铺垫：
现代策略模型不是凭空出现，而是在尝试用更强的条件分布建模能力，
解决动作多模态、长时域、离线数据覆盖不足等问题。
```

第三篇核心任务是把读者从“专家动作标签”带到“专家行为分布 / 奖励 / 数据覆盖”的层面。

## 二、第三篇导读改造计划

当前第三篇导读较短，只说明了 IRL、GAIL、Offline IL 的大概关系。建议扩写为以下结构：

```markdown
# 第三篇导读：经典模仿学习的分布匹配与奖励视角

## 1. 为什么第二篇之后需要第三篇

第二篇已经说明：
- 策略不是单步预测器；
- 策略闭环执行会诱导轨迹分布；
- 真正任务目标往往是轨迹级目标；
- 单步动作 loss 不能完全代表闭环成功。

但问题还没有结束：
如果单步动作拟合不够，我们应该匹配什么？

## 2. 第三篇的数学主线

单步动作拟合
→ 轨迹级行为解释
→ 奖励函数
→ reward ambiguity
→ feature expectation
→ maximum entropy IRL
→ occupancy measure
→ adversarial distribution matching
→ offline dataset support
→ OOD action 与失败恢复

## 3. 本篇新增数学对象

| 数学对象 | 符号 | 一句话含义 | 工程对应物 | 后续作用 |
|---|---|---|---|---|
| 奖励函数 | r(s,a) | 描述状态动作好坏的标量函数 | 安全、平顺、成功率、碰撞风险等偏好 | IRL / RL / 后训练 |
| 轨迹回报 | R(tau) | 一整条轨迹累计得到的 reward | 一次抓取或一次泊车全过程质量 | 连接轨迹目标 |
| 特征期望 | mu(pi) | 策略访问到的特征平均值 | 轨迹中安全距离、速度、成功状态等统计 | IRL 特征匹配 |
| 占用测度 | rho_pi(s,a) | 策略闭环执行时访问状态动作的频率 | 模型实际会走到哪里、做什么 | GAIL / 分布匹配 |
| 行为策略 | beta(a|s) | 产生离线数据的策略 | 遥操作员、旧模型、规则系统、混合数据源 | Offline IL |
| 支撑集 | support(D) | 数据覆盖到的状态动作区域 | 训练集中见过的正常、失败、恢复场景 | OOD 风险分析 |

## 4. 本篇读完后应该形成的工程判断

- IRL 不是简单“拟合 reward”，而是在处理专家行为背后的目标解释问题；
- reward 有歧义，同一条轨迹可能被很多 reward 解释；
- GAIL 的关键不是 GAN 类比，而是 occupancy measure matching；
- Offline IL 的核心不是数据量，而是 support 是否覆盖部署时会遇到的状态；
- 成功轨迹很多，不等于策略会恢复失败；
- 第四篇的 ACT / Diffusion Policy / Flow Matching 是更强策略表达，不是替代数据覆盖问题的魔法。
```

## 三、第10章 IRL 改造计划

### 10.1 改造定位

第10章应该回答：

```text
第二篇已经有轨迹分布和轨迹目标，
但还没有解释“专家为什么选择这些轨迹”。

IRL 引入 reward，
试图用 reward 解释专家行为背后的偏好结构。
```

不要把第10章写成“IRL 算法综述”，而要写成：

```text
从轨迹分布目标
→ 到专家偏好
→ 到 reward 函数
→ 到 reward ambiguity
→ 到最大熵 IRL
→ 到 GAIL 为什么可以绕开显式 reward
```

### 10.2 必须补强的数学对象

| 数学对象 | 建议处理方式 |
|---|---|
| reward function `r(s,a)` | 正式定义：不是标签，而是解释行为偏好的函数 |
| trajectory return `R(tau)` | 说明为什么 reward 要沿时间累积 |
| linear reward `r_w(s,a)=w^T f(s,a)` | 解释 feature、weight 与偏好之间的关系 |
| feature expectation `mu(pi)` | 作为 IRL 从轨迹统计解释偏好的关键对象 |
| reward ambiguity | 必须写成命题或专门小节 |
| maximum entropy trajectory distribution | 解释为什么引入熵，而不是只找一个最优 reward |

### 10.3 建议补充的核心公式主线

```text
第二篇得到轨迹分布 p_pi(tau)
→ 现在希望解释专家为什么偏好某些轨迹
→ 引入 reward r(s,a)
→ 一条轨迹的回报 R(tau)=sum_t r(s_t,a_t)
→ 如果 reward 是线性特征组合 r_w(s,a)=w^T f(s,a)
→ 则策略的偏好可以通过 feature expectation mu(pi) 表达
→ 但 reward 不唯一，引出 reward ambiguity
→ 最大熵 IRL 用 p_w(tau) ∝ exp(R_w(tau)) 建模专家轨迹概率
→ 梯度形式变成专家特征期望 - 模型特征期望
→ 这自然引出 GAIL：能不能直接匹配分布，而不是显式恢复 reward？
```

### 10.4 必须补的命题 / 证明

#### 命题 10.1：同一专家轨迹不能唯一确定 reward

说明：

```text
如果一个 reward 能解释专家轨迹，
那么对 reward 做常数平移、正比例缩放，甚至某些 potential-based transformation，
可能仍然不改变最优行为排序。
```

工程含义：

```text
不要以为 IRL 学到的 reward 就是真实专家内心目标。
它更像是一种行为解释函数，而不是物理真理。
```

#### 命题 10.2：线性 reward 下，策略比较可以转化为特征期望比较

核心推导：

```text
r_w(s,a)=w^T f(s,a)

R_w(tau)=sum_t w^T f(s_t,a_t)

E_pi[R_w(tau)]
= w^T E_pi[sum_t f(s_t,a_t)]
= w^T mu(pi)
```

工程含义：

```text
专家不一定每一步动作都要完全一样，
但如果安全距离、平顺性、目标对齐等统计特征接近，
就可能表现出相似偏好。
```

#### 命题 10.3：最大熵 IRL 的梯度是专家特征期望减模型特征期望

这是第10章最应该补强的推导。必须把：

```text
log likelihood
→ partition function
→ gradient
→ expert feature expectation - model feature expectation
```

完整讲清楚。

### 10.5 第10章接力修正

现有第10章中若出现“承接第11章”的表述，应改为：

```text
承接前文：承接第二篇关于轨迹分布和长期目标的讨论，尤其是第6章的轨迹级目标与第9章的概率建模视角。
本章推进：从“专家做了什么”进一步追问“什么 reward 能解释专家为什么这样做”。
铺垫后文：第11章将说明，如果显式恢复 reward 存在歧义和求解困难，我们可以改为直接匹配专家与策略的 occupancy measure。
```

## 四、第11章 GAIL 改造计划

### 11.1 改造定位

第11章应该回答：

```text
第10章说明了 IRL 可以通过 reward 解释专家行为，
但 reward 存在歧义，求解也复杂。

GAIL 换一个角度：
不显式恢复 reward，
而是直接让学习策略诱导出的 occupancy measure 接近专家。
```

### 11.2 必须补强的数学对象

| 数学对象 | 建议处理方式 |
|---|---|
| occupancy measure `rho_pi(s,a)` | 正式定义，并连接第二篇的轨迹分布 |
| discriminator `D_omega(s,a)` | 解释它不是动作预测器，而是分布判别器 |
| adversarial objective | 讲清楚 min-max 目标中的两个玩家 |
| implicit reward | 说明判别器输出如何变成策略优化信号 |
| entropy regularization | 解释为什么避免策略过早坍缩 |
| JS divergence / distribution matching | 可用“直觉 + 附录提示”方式讲，不必过度数学化 |

### 11.3 建议补充的核心公式主线

```text
第10章：IRL 试图学习 reward r(s,a)
→ 但 reward 歧义且求解复杂
→ GAIL 直接比较专家和策略的 rho_pi(s,a)
→ 判别器 D(s,a) 学会区分专家样本和策略样本
→ 策略更新目标是让判别器分不出来
→ min_pi max_D 形成对抗分布匹配
→ 判别器输出可转化为隐式 reward
→ 但 GAIL 需要 rollout 和交互，因此真实机器人部署困难
→ 下一章转入 Offline IL：如果不能在线 rollout，数据覆盖会变成核心问题
```

### 11.4 必须补的命题 / 证明

#### 命题 11.1：occupancy measure 比单步 action loss 更接近闭环行为

证明思路：

```text
BC 训练目标只在专家数据分布上比较 a_E 和 pi_theta(s)
GAIL 比较的是策略闭环执行诱导出的 rho_pi(s,a)
rho_pi 同时包含状态访问频率和动作选择
因此它比单步 loss 更能描述整体行为分布
```

工程含义：

```text
两个策略单步动作误差接近，不代表闭环轨迹相似；
但如果 occupancy measure 接近，说明它们常去的状态和常做的动作都接近。
```

#### 命题 11.2：最优判别器反映专家分布和策略分布的密度比

可以写成简化版，不需要把 GAN 证明写得太重。

核心目标：

```text
让读者知道：
判别器不是神秘奖励函数，
它本质上在估计两个分布哪里不一样。
```

#### 命题 11.3：GAIL 解决分布匹配问题，但不能绕开交互成本

工程判断：

```text
GAIL 理论上优雅，但通常需要当前策略 rollout；
真实机器人上 rollout 有安全、成本、设备损耗和数据回收问题。
```

### 11.5 第11章接力修正

建议把第11章中的倒序引用统一改为：

```text
承接前文：承接第10章 IRL 的 reward 学习视角，说明可以不显式恢复 reward，而是直接匹配专家与策略的 occupancy measure。
本章推进：把模仿学习从 reward inference 推进到 adversarial distribution matching。
铺垫后文：第12章将讨论，当我们不能让策略频繁在线 rollout，只能依赖离线数据时，数据覆盖、support mismatch 和 OOD action 会成为核心风险。
```

## 五、第12章 Offline Imitation Learning 改造计划

### 12.1 改造定位

第12章应该回答：

```text
第11章 GAIL 说明了在线 rollout 可以帮助匹配专家分布，
但真实机器人很难随便试错。

如果训练只能依赖离线数据，
那么模仿学习的核心风险就变成：
数据是否覆盖部署时可能进入的状态动作区域。
```

### 12.2 必须补强的数学对象

| 数学对象 | 建议处理方式 |
|---|---|
| offline dataset `D` | 说明它不是完整世界，只是采集策略留下的样本 |
| behavior policy `beta(a|s)` | 说明数据由谁产生，决定数据分布 |
| dataset distribution | 区分数据分布和部署策略分布 |
| support | 用图和例子解释数据覆盖区域 |
| support mismatch | 作为离线模仿失败的核心数学对象 |
| OOD action | 说明动作出圈会导致状态继续出圈 |
| recovery data | 作为工程闭环重点，而不是额外数据 |

### 12.3 建议补充的核心公式主线

```text
GAIL 需要当前策略 rollout 来估计 rho_pi(s,a)
→ 但真实机器人 rollout 成本高、风险高
→ Offline IL 只能使用固定数据集 D
→ D 由 behavior policy beta 产生
→ 因此 D 覆盖的是 rho_beta，而不是任意 rho_pi
→ 如果 pi_theta 进入 support(D) 外的状态动作区域
→ 模型预测将变成外推
→ 外推动作又会改变未来状态
→ OOD action 造成状态继续 OOD
→ 所以离线模仿学习的关键不是数据总量，而是 support 覆盖、失败恢复和边界样本
→ 第四篇的现代策略模型要在这个离线数据现实下进行条件动作分布建模
```

### 12.4 必须补的命题 / 证明

#### 命题 12.1：离线数据多，不等于闭环部署安全

证明思路：

```text
离线 loss 只在数据分布上估计；
如果部署策略诱导分布不同，
则离线 loss 不能保证 OOD 区域行为正确。
```

工程含义：

```text
一百万条顺利轨迹，可能不如少量高质量失败恢复轨迹对安全部署更有价值。
```

#### 命题 12.2：OOD action 会通过环境转移放大成 OOD state

核心链路：

```text
a_t outside support
→ s_{t+1} ~ P(s_{t+1}|s_t,a_t)
→ s_{t+1} 也可能 outside support
→ pi_theta 在 s_{t+1} 上继续外推
→ 闭环风险滚雪球
```

这条命题非常适合连接第3章分布偏移、第6章轨迹分布和第12章离线风险。

#### 命题 12.3：恢复轨迹覆盖的是“偏差后的可控区域”

工程判断：

```text
恢复轨迹不是脏数据；
它们是教策略如何从小偏差回到专家分布的关键数据。
```

### 12.5 第12章接力修正

建议改为：

```text
承接前文：承接第11章 GAIL 对策略诱导分布的关注，但指出真实机器人往往不能频繁在线 rollout。
本章推进：把模仿学习的核心风险转向离线数据覆盖、support mismatch、OOD action 和 recovery data。
铺垫后文：第四篇将讨论 ACT、Diffusion Policy、Flow Matching 等现代策略模型。它们提供更强的动作序列和多模态条件分布建模能力，但仍然不能绕开离线数据覆盖问题。
```

